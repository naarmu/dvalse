TC_DSH_RMS_004

서버 상태 변경 시 Dashboard 실시간 반영

Functional High
01

Precondition

  1. Dashboard 오픈 상태
02

Test Steps

  1. 다른 클라이언트에서 장비 상태 변경 (Online → Offline)
  2. Dashboard 수치가 자동 갱신되는지 확인
  3. Refresh 버튼 동작 확인 (우측 상단)
03

Test Data

장비 상태 변경 이벤트 발생
04

Expected Result

  1. Dashboard가 변경 사항을 일정 주기 내(30초 이내) 반영
  2. Refresh 버튼 클릭 시 즉시 갱신
  3. 갱신 시각이 어딘가에 표시됨 (선택)
05

비고 (QA · 개발 메모)

갱신 방식: Push(Server → Client) 방식이 이상적. Polling이라면 주기 확인.

06

제안 사항

UX 개선 마지막 Refresh 시각을 Dashboard 어딘가(예: 우측 상단)에 표시하면 사용자가 데이터 신선도를 판단할 수 있습니다.
