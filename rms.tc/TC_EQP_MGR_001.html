TC_EQP_MGR_001

Equipment Manager 진입 및 장비 목록 로딩

Functional Critical

Metadata

Area EQP · Equipment
Screen Equipment Manager
Sub-category 화면 진입
Test Type Functional
Priority Critical
Applicable Roles Vendor, Admin, Engineer, Operator
Estimated Time 2 min
Last Updated 2026.04.24

01Precondition

  1. RMS Client가 설치된 PC에서 정상적으로 프로그램이 실행됨
  2. Vendor 또는 Admin 계정으로 로그인되어 있음
  3. RMS 서버에 장비가 최소 1대 이상 등록되어 있음
  4. 등록된 장비 중 1대 이상이 Online 상태(Agent 정상 연결)

02Test Steps

  1. 좌측 메뉴에서 Equipment 클릭
  2. Equipment Manager 하위 메뉴 클릭
  3. 화면이 완전히 로드될 때까지 대기 (최대 5초)
  4. 좌측 장비 트리, 상단 툴바, 우측 상세 영역이 모두 표시되는지 확인
  5. 등록된 장비가 트리에 누락 없이 표시되는지 확인

03Test Data

Equipment: - EQ_001 (Online, Group: GROUP_A) - EQ_002 (Offline, Group: GROUP_A) - EQ_003 (Online, Group: 미지정) Login Account: admin_test / Admin Role

04Expected Result

  1. Equipment Manager 화면이 5초 이내에 표시됨
  2. 좌측 장비 트리에 등록된 모든 장비(EQ_001, EQ_002, EQ_003)가 누락 없이 표시됨
  3. 각 장비별 연결 상태 아이콘(Online / Offline)이 실제 상태와 일치하게 표시됨
  4. 상단 툴바/액션 버튼이 현재 Role(Admin) 권한에 맞게 표시됨 (Privilege 설정에 따름)
  5. 로딩 중 에러 메시지나 빈 화면이 나타나지 않음

05비고 (QA/개발 메모)

DevExpress 컴포넌트: 장비 트리는 TreeListControl, 우측 상세는 GridControl로 구성된 것으로 추정됨. 트리의 펼침/접힘 초기 상태 및 열 너비 기본값을 함께 확인할 것.

재현 주의: 화면 진입 시 백엔드 API 호출이 2회 이상 발생할 경우 체감 로딩 시간이 늘어날 수 있음 — 네트워크 탭으로 호출 횟수 확인 권장.

06제안 사항

UX 개선 장비 목록이 많아질 경우(50대 이상) 트리의 기본 펼침 상태가 UX를 저해할 수 있습니다. 기본적으로 그룹 단위까지만 펼쳐두고, 장비는 접힘 상태로 시작하는 기본값을 제안드립니다.

기획 검토 현재 Precondition에서 "Vendor/Admin" 전용으로 가정했지만, Operator가 Equipment Manager에 조회만 가능한 형태로 접근할 수 있어야 하는지 Privilege 기본값 정의가 필요합니다. 정의 후 본 TC의 Applicable Roles를 확정하겠습니다.
