URS_SET_PRF

개인 설정

User Requirement Configuration
01

기능 개요

사용자가 RMS Client의 표시 환경(언어, 테마, 단위 등)을 자신에게 맞춰 설정하는 화면이다. 여기서 설정한 값은 해당 사용자에게만 적용되며, 다른 사용자나 시스템 전체에는 영향을 주지 않는다.

Preference는 운영 데이터에 영향을 주지 않는 표시 옵션의 모음이다. 시스템 동작 자체나 데이터 무결성과 관련된 설정은 별도 화면(Connection, Privilege, Account)에서 관리한다.

02

사용자 니즈

주요 사용자

  • RMS의 모든 사용자 — 자신에게 익숙한 표시 환경에서 작업

Pain Point (RMS 도입 전)

  • 기본 언어·단위가 사용자 환경과 맞지 않아 가독성 저하
  • 장시간 사용 시 눈의 피로 누적 (테마 옵션 부재)

기대 효과

  • 사용자별로 익숙한 표시 환경에서 작업 가능
  • 다크 테마 지원으로 장시간 사용 시 피로도 감소
  • 설정 보존으로 매 로그인마다 재설정 불필요
03

기능 요구사항

■ FR-1. 설정 항목

  • FR-1.1 사용자는 표시 언어(한국어 / English 등)를 변경할 수 있어야 한다.
  • FR-1.2 사용자는 테마(Light / Dark)를 변경할 수 있어야 한다.
  • FR-1.3 사용자는 단위 표시 형식(예: 길이, 시간 형식, 숫자 구분자)을 변경할 수 있어야 한다.
  • FR-1.4 사용자는 Dashboard 자동 갱신 주기 등 부가 설정을 변경할 수 있다.

■ FR-2. 저장 · 적용

  • FR-2.1 사용자는 변경한 설정을 명시적 저장 동작으로 영구 반영할 수 있어야 한다.
  • FR-2.2 일부 설정(테마 등)은 저장 즉시 화면에 반영되며, 일부 설정(언어 등)은 재시작 또는 화면 갱신 시 반영된다는 안내를 표시한다.
  • FR-2.3 저장된 설정은 사용자가 RMS를 재접속해도 유지된다.

■ FR-3. 입력값 검증

  • FR-3.1 숫자 입력 항목(예: 갱신 주기)에 잘못된 값(범위 밖, 형식 오류) 입력 시 저장을 차단한다.
  • FR-3.2 검증 실패 시 어느 항목의 어떤 값이 문제인지 명확한 에러 메시지를 표시한다.
  • FR-3.3 검증은 사용자가 입력하는 도중 실시간으로 수행되어야 한다.

■ FR-4. 기본값 복원

  • FR-4.1 사용자는 모든 설정을 시스템 기본값으로 일괄 초기화할 수 있어야 한다.
  • FR-4.2 초기화 전 사용자에게 확인을 받는다.
  • FR-4.3 초기화 직전의 설정을 Export하여 백업할 수 있는 옵션을 제공한다.

■ FR-5. 사용자 범위

  • FR-5.1 Preference 설정은 해당 사용자에게만 적용된다.
  • FR-5.2 다른 사용자나 시스템 전체 설정에는 영향을 주지 않는다.
  • FR-5.3 사용자가 다른 PC에서 같은 계정으로 접속해도 설정이 동일하게 적용된다(서버 측 보존).
04

비기능 요구사항

■ NFR-1. 권한

  • NFR-1.1 Preference는 모든 사용자(All Roles)가 자신의 설정을 변경할 수 있다.
  • NFR-1.2 다른 사용자의 Preference에는 어느 Role도 접근할 수 없다.

■ NFR-2. 사용성

  • NFR-2.1 즉시 반영되는 설정과 재시작이 필요한 설정의 구분이 사용자에게 명확히 안내되어야 한다.

■ NFR-3. 데이터 보존

  • NFR-3.1 설정값은 서버에 보관되어 사용자가 다른 PC에서 접속해도 동일하게 적용된다.
05

제약사항 · 정책

  • 정책 Preference는 사용자별 개인 설정이다. 다른 사용자나 시스템 전체에 영향을 주지 않는다.
  • 정책 설정값은 서버에 보관되어 다른 PC에서 동일 계정 접속 시에도 적용된다.
  • 정책 설정 변경의 적용 시점(즉시 / 재시작 시)이 사용자에게 명확히 안내되어야 한다.
  • 정책 기본값 복원 전 현재 설정을 Export로 백업할 수 있다.
  • 제약 Preference 설정은 데이터 무결성에 영향을 주지 않는 표시 옵션에 한정된다. 시스템 동작 관련 설정(연결 정보 등)은 별도 화면에서 관리한다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 5건:

  • TC_SET_PRF_001 — Preference 화면 진입 및 설정값 로드 (FR-1)
  • TC_SET_PRF_002 — 설정 변경 · 저장 후 재접속 시 유지 (FR-2.3, NFR-3.1)
  • TC_SET_PRF_003 — 언어 · 테마 · 단위 등 기본 환경 설정 동작 (FR-1.1, FR-1.2, FR-1.3, FR-2)
  • TC_SET_PRF_004 — 잘못된 값 입력 시 검증 (FR-3)
  • TC_SET_PRF_005 — 설정 초기화 (Reset to Default) (FR-4)
