TC_EQP_MGR_007

장비 연결 상태 실시간 표시

Functional Critical
01

Precondition

  1. Equipment Manager 화면 진입 완료
  2. 장비 EQ_001이 Online 상태로 표시됨
  3. 해당 장비 PC의 RMS Agent 프로세스가 실제 실행 중
02

Test Steps

  1. EQ_001의 상태가 Online으로 표시되는 것을 확인
  2. EQ_001 장비 PC에서 RMS Agent 프로세스 종료 (또는 네트워크 케이블 분리)
  3. 일정 시간 내(예: 15초 이내) UI에서 상태가 Offline으로 전환되는지 확인
  4. Agent 재시작 (또는 네트워크 재연결)
  5. 일정 시간 내 상태가 Online으로 복귀하는지 확인
  6. RMS Dashboard의 해당 장비 카드에도 상태 변경이 반영되는지 확인
03

Test Data

Equipment: EQ_001 Initial Status: Online Trigger 1: Agent 종료 → Offline 전환 예상 Trigger 2: Agent 재시작 → Online 복귀 예상
04

Expected Result

  1. Agent 종료 후 15초 이내 UI에서 Offline으로 상태 전환
  2. Agent 재시작 후 15초 이내 UI에서 Online으로 복귀
  3. 상태 전환 시 아이콘 색상/모양이 즉시 변경됨
  4. 화면을 새로고침하지 않아도 자동 갱신됨 (Push 또는 Polling 방식 무관)
  5. Dashboard 화면에서도 동일하게 상태가 동기화됨
  6. 상태 변경 이력이 Equipment History에 기록됨
05

비고 (QA · 개발 메모)

갱신 주기: Polling 방식일 경우 주기를 확인하고, 주기에 따라 허용 가능한 지연 시간을 결정해야 함. Push 방식이라면 WebSocket/SignalR 연결 끊김 시 재연결 동작 확인 필요.

네트워크 경계 케이스: Agent는 살아있으나 네트워크만 단절된 경우 vs. 장비 PC 자체가 꺼진 경우 구분 가능한지 확인.

06

제안 사항

기획 검토 Offline 판정 임계 시간(Threshold)이 정의되어 있어야 합니다. 순간적인 네트워크 끊김으로 상태가 깜빡거리는 것을 방지하기 위해 연속 N회 미응답 시 Offline 처리 같은 히스테리시스 정책을 권장드립니다.

UX 개선 상태 전환 시 미묘한 트랜지션 애니메이션(예: 아이콘 색상 0.3초 페이드)을 주면, 사용자가 "상태가 방금 바뀌었다"는 것을 인지하기 쉽습니다.

데이터 무결성 상태 변경 이력이 Equipment History에 모든 전환마다 기록되는지, 아니면 사용자가 해당 화면을 볼 때만 기록되는지 정책 확인 필요. 반도체 공정 환경에서는 감사(Audit) 관점에서 모든 전환 기록을 권장합니다.
