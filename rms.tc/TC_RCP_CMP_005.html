TC_RCP_CMP_005

같은 Recipe의 버전 간 비교

Functional High
01

Precondition

  1. Recipe "213"이 v1, v2 두 버전을 가짐
  2. 두 버전 간 파라미터 값이 일부 다름
02

Test Steps

  1. Compare Recipes에서 첫 슬롯에 Recipe 213 / v1 선택
  2. 두 번째 슬롯에도 Recipe 213을 선택하되 버전은 v2 지정
  3. 비교 결과에서 두 버전 간 차이 확인
  4. 동일 Recipe라는 점이 UI에 적절히 표시되는지 확인 (예: "Recipe 213 v1 ↔ v2")
03

Test Data

Recipe 213: v1(Die X=20.0) / v2(Die X=25.0)
04

Expected Result

  1. 같은 Recipe의 다른 버전을 비교 대상으로 선택 가능
  2. 비교 테이블 헤더에 버전 정보가 명확히 구분 표시
  3. 변경된 파라미터만 하이라이트
  4. v1 → v2 방향성이 UI에 드러남 (왼→오 등)
05

비고 (QA · 개발 메모)

주 용도: 특정 Recipe의 변경 이력을 추적할 때 가장 많이 사용되는 기능. Recipe Manager에서 버전 선택 → Compare 진입 시 자동 세팅되면 좋음.

06

제안 사항

UX 개선 Recipe Manager의 버전 노드를 우클릭 → "이 버전을 이전 버전과 비교" 퀵 메뉴를 제공하면 반복 작업이 크게 편해집니다.

데이터 무결성 버전 간 비교 결과를 CSV/Excel로 Export할 수 있으면 공정 검토 회의 자료로 활용도가 높아집니다.
