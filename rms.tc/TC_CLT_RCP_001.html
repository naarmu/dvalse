TC_CLT_RCP_001

수집된 Recipe가 Pending Recipes에 표시

Functional Critical
01

Precondition

  1. 장비가 Online
  2. 장비에서 Recipe 파일 생성 또는 수정
02

Test Steps

  1. 장비에서 Recipe 파일 변경 발생
  2. Agent가 변경 감지하고 서버로 보고
  3. RMS UI의 Pending Recipes에 항목 추가 확인
03

Test Data

장비에서 새 Recipe 생성
04

Expected Result

  1. Pending 목록에 신규 항목 등록
  2. 장비명, Recipe명, 변경유형 표시
05

비고 (QA · 개발 메모)

감지 주기: Agent가 파일 변경을 실시간 감지인지, 주기적 스캔인지 확인.

06

제안 사항

기획 검토 파일 변경 감지 방식을 파일 시스템 이벤트 기반(실시간)으로 구현하면 지연이 최소화됩니다.
