URS_HST_MTD

Method 이력

User Requirement Audit
01

기능 개요

Method와 관련된 모든 이벤트(등록·수집·배포·삭제·변경 등)의 시계열 이력을 조회하는 화면이다. Recipe History와 동일한 운영 모델을 따르되 대상이 Method라는 점만 다르다.

Method는 Recipe보다 운영 빈도가 낮지만, 이력 추적의 중요성은 동일하다. Method 변경이 공정 결과에 영향을 줄 수 있으므로 모든 변경이 추적 가능해야 한다.

02

사용자 니즈

주요 사용자

  • 공정 표준화 팀 — Method 변경 사유와 시점 추적
  • 장비 엔지니어 — 특정 Method의 변경 이력 검토
  • 감사 담당자 — Method 운영 이력 검증

Pain Point (RMS 도입 전)

  • Method 변경이 언제, 누가, 왜 이루어졌는지 기록되지 않음
  • Method가 Recipe보다 더 자주 변경되는데도 이력 관리가 더 부족함

기대 효과

  • Method 모든 변경의 자동 기록과 시계열 조회
  • Recipe History와 동일한 UX로 학습 비용 최소화
03

기능 요구사항

■ FR-1. 이력 목록 조회

  • FR-1.1 화면 진입 시 Method 관련 이벤트 이력이 시계열(최신 우선)로 표시된다.
  • FR-1.2 각 항목에는 발생 시각, Method명, 버전, 이벤트 유형, 사용자, 상세 내용이 표시된다.
  • FR-1.3 이벤트 유형은 Register / Collected / Deployed / Updated / Deleted를 포함한다.

■ FR-2. 필터링

  • FR-2.1 사용자는 날짜 범위, 이벤트 타입, 사용자, Method명으로 필터링할 수 있어야 한다.
  • FR-2.2 다중 필터를 동시 적용할 수 있다(AND 조건).

■ FR-3. Method별 이력 추적

  • FR-3.1 사용자는 특정 Method를 선택하여 그 Method의 모든 이력만 추출할 수 있어야 한다.
  • FR-3.2 Method Manager의 상세 화면에서 "이력 보기" 진입점을 제공한다.

■ FR-4. Export

  • FR-4.1 사용자는 현재 필터링 결과를 CSV 또는 Excel로 Export할 수 있어야 한다.
  • FR-4.2 Export 파일은 SEMI 감사 대응 포맷에 맞춰 컬럼 구성과 메타가 포함된다.

■ FR-5. 데이터 보존

  • FR-5.1 이력 데이터는 어떤 사용자도 수정·삭제할 수 없다(append-only).
  • FR-5.2 이력 보존 기간은 최소 3년이다.
  • FR-5.3 삭제된 Method의 이력도 보존되며 조회 가능해야 한다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 화면 진입 후 초기 데이터 표시까지 5초 이내.
  • NFR-1.2 필터 적용 응답은 3초 이내.

■ NFR-2. 권한

  • NFR-2.1 이력 조회는 모든 사용자(All Roles)가 가능하다.
  • NFR-2.2 Export는 Vendor, Admin, Engineer가 가능하다.

■ NFR-3. UI 일관성

  • NFR-3.1 Method History는 Recipe History와 동일한 UI·UX 패턴을 따른다.
05

제약사항 · 정책

  • 정책 모든 이력은 append-only로 관리된다.
  • 정책 이력 보존 기간은 최소 3년이다.
  • 정책 Method가 삭제되어도 그 이력은 보존된다.
  • 정책 Recipe History와 동일한 UI 패턴을 따른다. 학습 비용 최소화가 목적이다.
  • 제약 Export는 Vendor, Admin, Engineer로 한정된다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 4건:

  • TC_HST_MTD_001 — Method 이력 조회 (등록 · 수집 · 배포 · 변경) (FR-1)
  • TC_HST_MTD_002 — 이벤트 타입별 필터링 (FR-2)
  • TC_HST_MTD_003 — Method별 이력 추적 (FR-3)
  • TC_HST_MTD_004 — 이력 Export 기능 (FR-4)
