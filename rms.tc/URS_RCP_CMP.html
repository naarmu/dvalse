URS_RCP_CMP

Recipe 비교

User Requirement Core Function
01

기능 개요

두 개 이상의 Recipe 또는 같은 Recipe의 서로 다른 버전을 나란히 놓고 파라미터 값의 차이를 한눈에 확인하는 기능이다. 사용자는 이 화면을 통해 어떤 파라미터가 어떻게 다른지 즉시 파악하고, 의도된 변경인지 검토한다.

Recipe Manager가 개별 Recipe의 상태를 보여준다면, Compare Recipes는 여러 Recipe 사이의 관계를 보여주는 화면이다. 공정 검토, 변경 추적, 표준 일치 여부 확인 등 다양한 운영 시나리오의 출발점이 된다.

02

사용자 니즈

주요 사용자

  • 공정 엔지니어 — Recipe 변경 전후 파라미터 차이 검토
  • 장비 엔지니어 — 같은 Recipe인데 장비별로 값이 다른지 확인
  • 공정 표준화 팀 — 표준 Recipe 대비 개별 장비의 일치 여부 확인

Pain Point (RMS 도입 전)

  • 두 Recipe의 파라미터를 일일이 출력하거나 화면에 띄워놓고 눈으로 비교
  • 파라미터 수가 많을수록 사람이 놓치는 차이가 발생
  • 버전 간 변경점을 사후에 추적하기 어려움

기대 효과

  • 파라미터 차이가 시각적으로 즉시 드러남
  • 버전 변경 검토 시간 단축
  • 공정 표준 준수 확인이 일상적인 작업으로 가능
03

기능 요구사항

■ FR-1. 비교 대상 선택

  • FR-1.1 사용자는 비교할 Recipe와 버전을 슬롯 단위로 선택할 수 있어야 한다.
  • FR-1.2 같은 Recipe의 서로 다른 버전을 비교 대상으로 선택할 수 있어야 한다.
  • FR-1.3 Recipe Manager에서 Recipe를 선택한 상태로 Compare 진입 시, 선택된 Recipe가 첫 번째 슬롯에 자동 입력된다.
  • FR-1.4 동일 Recipe의 동일 버전이 두 슬롯에 중복 선택되는 경우 차단하거나 경고한다.

■ FR-2. 파라미터 비교 표시

  • FR-2.1 비교 대상이 선택되면 파라미터를 행으로, 각 Recipe를 컬럼으로 하는 테이블이 표시되어야 한다.
  • FR-2.2 테이블 헤더에는 Recipe 이름과 버전 정보가 함께 표시된다.
  • FR-2.3 Manage Recipe Parameters에서 숨김 설정된 파라미터는 비교 테이블에서도 표시되지 않는다.
  • FR-2.4 표시되는 파라미터 외에 숨겨진 파라미터가 있는 경우, 그 개수를 화면 상단에 안내한다.

■ FR-3. 불일치 강조

  • FR-3.1 행 내 모든 값이 같은 파라미터는 일반 표시한다.
  • FR-3.2 행 내 하나라도 다른 값이 있는 파라미터는 배경색 하이라이트로 강조 표시한다.
  • FR-3.3 한 Recipe에만 존재하고 다른 Recipe에는 없는 파라미터는 별도로 구분 표시한다.
  • FR-3.4 색상에 더해 아이콘(예: ⚠ 또는 ●)을 병행하여 색각 이상 사용자도 차이를 인지할 수 있어야 한다.
  • FR-3.5 사용자는 "차이만 보기" 토글을 통해 일치 행을 숨기고 변경된 파라미터만 볼 수 있어야 한다.

■ FR-4. 다중 비교

  • FR-4.1 사용자는 3개 이상 5개 이하의 Recipe를 동시에 비교할 수 있어야 한다.
  • FR-4.2 다중 비교 시 한 행 내 값이 모두 같으면 일반, 하나라도 다르면 하이라이트로 표시한다.
  • FR-4.3 다중 비교 시 사용자는 기준 Recipe를 지정할 수 있으며, 지정된 기준과 다른 값만 하이라이트할 수 있어야 한다.

■ FR-5. 결과 활용

  • FR-5.1 사용자는 비교 결과를 CSV 또는 Excel로 Export할 수 있어야 한다.
  • FR-5.2 Recipe Manager의 버전 노드에서 우클릭으로 "이 버전을 이전 버전과 비교" 액션을 제공한다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 2개 Recipe 비교 결과 표시까지 1.5초 이내.
  • NFR-1.2 5개 Recipe 다중 비교 시에도 3초 이내.
  • NFR-1.3 파라미터 100개 이상의 Recipe 비교에서도 스크롤 끊김이 없어야 한다.

■ NFR-2. 접근성

  • NFR-2.1 하이라이트 표시는 색상만으로 의미를 전달하지 않는다. 색상 + 아이콘 또는 색상 + 텍스트 라벨 병행을 원칙으로 한다.
  • NFR-2.2 하이라이트 색상은 다크 모드와 라이트 모드 모두에서 충분한 대비를 가져야 한다.

■ NFR-3. 사용성

  • NFR-3.1 비교 화면에서 다른 Recipe로 슬롯을 변경할 때 Recipe Manager로 돌아가지 않고 진행할 수 있어야 한다.
  • NFR-3.2 최근 비교 이력을 일정 기간 보관하여 재실행이 가능하도록 한다.
05

제약사항 · 정책

  • 정책 비교 대상 최대 개수는 5개이다. 화면 가로폭 가독성과 유의미한 비교 단위를 고려한 기준이다.
  • 정책 Manage Recipe Parameters에서 숨김 처리된 파라미터는 비교 결과에서도 동일하게 숨김 처리된다.
  • 정책 하이라이트는 색상 + 아이콘 병행으로 표시한다. 색각 이상 사용자(인구의 약 8%) 대응이 목적이다.
  • 정책 다중 비교 시 기준 Recipe를 지정한 경우, 기준과 다른 값만 하이라이트된다.
  • 제약 동일 Recipe의 동일 버전을 두 슬롯에 중복 선택할 수 없다.
  • 제약 비교 결과 Export는 모든 사용자가 가능하다. (조회 권한과 동일)
