TC_RCP_XML_004

배포된 XML이 장비에서 .psr로 자동 변환

Functional Critical
01

Precondition

  1. Custom XML이 EQ_001로 방금 배포 완료됨
  2. EQ_001의 RMS Agent가 정상 동작 중
02

Test Steps

  1. 배포 직후 EQ_001 장비 PC에서 Recipe 저장 경로 확인
  2. XML 파일이 .psr로 변환되어 저장되었는지 확인
  3. RMS Agent가 변환된 .psr을 서버로 수집하는지 확인
  4. Recipe Manager에서 새로 등록된 Recipe 조회 (TC_RCP_XML_005 연계)
03

Test Data

Deployed XML: custom_recipe.xml Expected PSR: custom_recipe.psr (장비 내 Recipe 폴더)
04

Expected Result

  1. XML 수신 후 Agent가 자동으로 .psr로 변환
  2. 변환된 .psr 파일이 장비의 정상 Recipe 경로에 저장
  3. 변환 성공 이력이 Server Log 또는 Equipment History에 기록
  4. 변환된 .psr은 기존 Recipe와 동일하게 장비에서 사용 가능
05

비고 (QA · 개발 메모)

핵심 기능: Waferless Recipe Creation의 최종 목적지. 변환 로직의 정확성이 매우 중요.

변환 주체: Agent가 변환하는지, 장비의 XEA Service가 변환하는지 아키텍처 확인.

06

제안 사항

데이터 무결성 XML → PSR 변환은 원자적(atomic)이어야 합니다. 변환 중 실패 시 중간 상태 파일이 남지 않도록 temp 파일 → rename 패턴 필수.

기획 검토 변환 실패 시 기존 동일명 Recipe를 덮어쓸지 여부에 대한 정책이 필요합니다. 권장: 변환 실패 시 기존 Recipe 보존 + 관리자에게 알림.
