TC_EQP_MGR_005

장비 선택 시 우측 상세 영역 갱신

UI High
01

Precondition

  1. Equipment Manager 화면 진입 완료
  2. 장비 3대 이상 존재 (서로 다른 Group, Status)
02

Test Steps

  1. 좌측 트리에서 EQ_001 선택
  2. 우측 상세 영역에 EQ_001 정보 표시 확인
  3. 빠르게 EQ_002로 선택 변경
  4. 우측 상세 영역이 EQ_002 정보로 즉시 갱신되는지 확인
  5. 선택을 빠르게 여러 번 변경했을 때 플리커(깜빡임)나 잘못된 정보 섞임이 없는지 확인
03

Test Data

EQ_001 (Online, GROUP_A), EQ_002 (Offline, GROUP_B), EQ_003 (Online, 미지정)
04

Expected Result

  1. 좌측 트리 선택이 변경될 때마다 우측 상세 영역이 선택된 장비 정보로 즉시 갱신됨 (1초 이내)
  2. 전환 중 이전 장비의 정보가 남아있거나 섞여 표시되지 않음
  3. 데이터 로딩 중일 경우 로딩 인디케이터가 적절히 표시됨
  4. 선택된 장비 행(Row)이 시각적으로 하이라이트됨
  5. 빠른 연속 클릭 시에도 최종 선택된 장비의 정보가 정확히 표시됨
05

비고 (QA · 개발 메모)

레이스 컨디션: 장비 A 선택 → 로딩 중 장비 B 선택 → A 응답이 뒤늦게 와서 B 상세에 A 정보가 덮어써지는 문제를 특히 주의 깊게 확인.

DevExpress: TreeListControl의 SelectionChanged 이벤트 처리 방식 확인 필요.

06

제안 사항

UX 개선 선택된 장비의 행 하이라이트 컬러가 Park Systems UI의 하늘색 계열과 일관되는지 확인 필요합니다.

성능 상세 영역 갱신이 매번 서버 API를 재호출하는 방식이라면, 최근 조회한 장비의 정보를 짧은 시간(예: 5초) 클라이언트 캐싱하는 것을 고려할 수 있습니다.
