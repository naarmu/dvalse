TC_RCP_MGR_007

로딩 중 상태 (Loading Spinner) 표시

UI Low
01

Precondition

  1. 대용량 Recipe(수백 개)가 서버에 등록된 환경
  2. Recipe Manager 진입 시 로딩이 체감되는 상태
02

Test Steps

  1. Recipe Manager로 화면 전환
  2. 데이터 로딩 중 로딩 인디케이터 표시 확인 (slide-28 참고)
  3. Recipe 선택 후 상세 로딩 중에도 인디케이터 동작 확인
  4. 네트워크 지연 발생 시 인디케이터가 즉시 나타나는지 확인
03

Test Data

Recipe 300+ (대용량), 네트워크 지연 시뮬레이션
04

Expected Result

  1. 로딩 시작 직후 300ms 이내에 스피너 또는 로딩 바 표시
  2. 로딩 완료 시 자연스럽게 제거
  3. 로딩 중에도 다른 UI 요소(메뉴, 헤더)는 조작 가능
  4. 매우 짧은 로딩(200ms 이내)에는 스피너를 띄우지 않아 깜빡임 방지
05

비고 (QA · 개발 메모)

slide-28 참고: 캡쳐 화면에 중앙에 원형 스피너가 표시됨. 동일 디자인·위치 기준.

Flash 방지: 너무 빠른 응답에 스피너가 깜빡거리는 현상은 UX에 부정적. 지연 표시(delayed show) 패턴 적용.

06

제안 사항

UX 개선 스피너 대신 스켈레톤 UI를 사용하면 사용자 체감 로딩 시간을 줄일 수 있습니다. 다만 Park Systems UI 톤과의 일관성을 고려해 결정.
