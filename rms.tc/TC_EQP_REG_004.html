TC_EQP_REG_004

필수 입력 누락 시 등록 차단

Negative High
01

Precondition

  1. Vendor 또는 Admin 계정으로 로그인됨
  2. Register Equipment 화면 진입
02

Test Steps

  1. 아무 필드도 입력하지 않고 Save 버튼 클릭
  2. 에러 메시지 확인
  3. Equipment ID만 입력하고 Save (나머지 필수 누락)
  4. Equipment ID + Name만 입력하고 Save (IP 누락)
  5. 모든 필수 필드 입력 후 Save → 정상 등록 확인
03

Test Data

Case 1: (전체 공란) Case 2: Equipment ID만 Case 3: Equipment ID + Name Case 4: 모든 필수 필드
04

Expected Result

  1. 필수 필드가 하나라도 누락되면 등록이 차단됨
  2. 누락된 필드에 대해 각각 에러 표시 (필드 하단 또는 Summary 영역)
  3. 에러 메시지는 어떤 필드가 누락되었는지 명확히 안내
  4. 필수 필드에는 UI상 * 표시 또는 "필수" 라벨이 있어야 함
  5. 모든 필수 필드 입력 후에는 정상 등록됨
05

비고 (QA · 개발 메모)

필수 필드 정의: Equipment ID, Name, IP가 필수로 가정됨. 기획 확정 시 조정.

에러 표시 방식: 개별 필드 에러 vs 상단 Summary 박스 방식 — UI 표준에 맞춰야 함.

06

제안 사항

UX 개선 필수 필드에 * 표시와 함께 placeholder 텍스트를 제공하여 사용자가 입력 전부터 필수 여부를 인지하도록 하면 에러를 사전에 줄일 수 있습니다.

UX 개선 Save 버튼을 필수 필드 미충족 시 비활성화 상태로 유지하는 것도 대안입니다. 단, 이 경우 "왜 버튼이 눌리지 않는지" tooltip으로 설명이 필요합니다.
