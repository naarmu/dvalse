URS_DSH_RCP

Recipe 운영 대시보드

User Requirement Overview
01

기능 개요

Recipe 자산의 운영 현황에 특화된 대시보드이다. 사용자는 이 화면에서 전체 Recipe 수, 최근 수집·배포 건수, Master 지정 현황, 최근 변경된 Recipe 목록 등을 한눈에 확인한다.

Recipe Manager가 개별 Recipe의 관리에 집중한다면, Recipe Dashboard는 Recipe 자산 전체의 흐름과 변화를 보여준다. 공정 표준화 팀이 Recipe 운영 추세를 파악하기 위한 화면이다.

02

사용자 니즈

주요 사용자

  • 공정 표준화 팀 — Recipe 운영 추세 파악
  • RMS 관리자 — Recipe 자산 현황 점검
  • 공정 엔지니어 — 최근 변경된 Recipe 빠른 확인

Pain Point (RMS 도입 전)

  • Recipe가 얼마나 수집·배포되었는지 추적하려면 별도 집계 필요
  • Master 미지정 Recipe가 누적되어도 인지하지 못함
  • 최근 변경 사항을 일별로 확인할 방법이 없음

기대 효과

  • Recipe 운영 활동을 시계열로 조감
  • Master 미지정 Recipe를 일별로 인지
  • 최근 수집·배포 활동의 빠른 확인
03

기능 요구사항

■ FR-1. 전체 Recipe 현황 위젯

  • FR-1.1 등록된 전체 Recipe 수, 최근 7일간 수집된 Recipe 수, 최근 7일간 배포 건수가 위젯에 표시된다.
  • FR-1.2 위젯의 수치를 클릭하면 Recipe Manager 또는 Deployment Log로 이동하며 해당 컨텍스트로 필터링된다.

■ FR-2. 수집 · 배포 추이

  • FR-2.1 최근 일정 기간(7일·30일 등) 동안의 수집·배포 건수 추이가 차트 형태로 표시된다.
  • FR-2.2 차트의 기간 선택은 사용자가 변경할 수 있어야 한다.
  • FR-2.3 수집과 배포의 두 지표가 시각적으로 구분되어 표시된다.

■ FR-3. Master 지정 현황

  • FR-3.1 Master가 지정된 Recipe 수와 미지정 Recipe 수가 위젯에 표시된다.
  • FR-3.2 Master 미지정 Recipe가 임계값 이상인 경우 시각 강조 표시한다.
  • FR-3.3 위젯 클릭 시 Master 미지정 Recipe 목록으로 이동한다.

■ FR-4. 최근 변경 Recipe

  • FR-4.1 최근 변경된 Recipe 목록이 시간순으로 표시된다.
  • FR-4.2 각 항목에는 Recipe명, 변경 유형(생성·버전 추가·삭제·Master 변경), 변경자, 변경 시각이 함께 표시된다.
  • FR-4.3 항목 클릭 시 해당 Recipe 상세로 이동한다.

■ FR-5. 실시간 갱신

  • FR-5.1 Dashboard의 모든 위젯은 일정 주기로 자동 갱신된다.
  • FR-5.2 사용자는 수동 새로고침으로 즉시 갱신할 수 있다.
  • FR-5.3 마지막 갱신 시각이 화면에 표시된다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 Dashboard 로딩은 5초 이내 완료되어야 한다.
  • NFR-1.2 Recipe 1,000개 이상 환경에서도 위젯 표시 성능이 유지되어야 한다.
  • NFR-1.3 집계는 사전 계산된 형태로 제공하여 실시간 부하를 최소화한다.

■ NFR-2. 권한

  • NFR-2.1 Recipe Dashboard는 모든 사용자(All Roles)가 조회 가능하다.

■ NFR-3. 데이터 신선도

  • NFR-3.1 위젯에 표시된 데이터의 마지막 갱신 시각이 사용자에게 보여야 한다.
05

제약사항 · 정책

  • 정책 Dashboard 자동 갱신 주기는 30초이다. (RMS Dashboard·Equipment Dashboard와 동일)
  • 정책 Master 미지정 Recipe가 임계값 이상인 경우 시각 강조한다. 운영 표준 관리 누락을 방지하기 위함이다.
  • 정책 위젯 클릭 시 해당 컨텍스트가 적용된 상세 화면으로 자동 이동한다.
  • 정책 집계 데이터는 사전 계산된 형태로 제공한다.
  • 제약 Recipe 1,000개 이상 환경에서도 5초 이내 로딩이 가능해야 한다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 5건:

  • TC_DSH_RCP_001 — Recipe Dashboard 전체 Recipe 현황 표시 (FR-1)
  • TC_DSH_RCP_002 — Recipe 수집 · 배포 건수 요약 정확성 (FR-1, FR-2)
  • TC_DSH_RCP_003 — Master Recipe 지정 현황 표시 (FR-3)
  • TC_DSH_RCP_004 — Recipe 데이터 갱신 시 Dashboard 반영 (FR-4, FR-5)
  • TC_DSH_RCP_005 — 대량 Recipe (1000+) 상태에서 로딩 성능 (NFR-1.2)
