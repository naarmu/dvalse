TC_EQP_GRP_008

다수 그룹 생성 시 목록 표시 성능

Non-func Low
01

Precondition

  1. 서버에 50개 이상의 그룹이 등록된 상태 (사전 세팅 필요)
  2. 각 그룹에 5대 이상의 장비가 소속된 상황
02

Test Steps

  1. Edit Equipment Groups 화면 진입
  2. 목록 로딩 시간 측정 (체감 및 개발자 도구)
  3. 그룹 1개 선택 시 상세 영역 갱신 시간 측정
  4. 그룹 추가/삭제 시 목록 갱신 시간 측정
  5. 스크롤 스무스함 확인
03

Test Data

Groups: 50+ 개, 각 그룹 멤버: 5대 이상
04

Expected Result

  1. 목록 초기 로딩 3초 이내
  2. 그룹 선택 시 상세 갱신 1초 이내
  3. 그룹 추가/삭제 후 목록 갱신 2초 이내
  4. 스크롤이 끊김 없이 매끄러움 (60fps 근처 유지)
  5. 메모리 사용량이 비정상적으로 증가하지 않음
05

비고 (QA · 개발 메모)

실전 운영 규모: Park Systems 고객사 기준 평균 그룹 수와 장비 수가 얼마인지 확인 필요. 50개는 여유를 둔 수치.

가상화(Virtualization): DevExpress TreeListControl의 Virtual Scrolling이 활성화되어 있는지 확인. 대량 데이터에서 필수.

06

제안 사항

성능 그룹/장비가 100개를 넘어가면 트리 가상화(Virtual Scrolling)가 필수가 됩니다. 현 시점에 확인·적용을 권장드립니다.

기획 검토 실제 운영 환경의 최대 그룹 수·장비 수 상한선을 기획적으로 정의해두는 것이 좋습니다. 예: "1 서버당 그룹 100개, 장비 500대까지 지원" 같은 SLA.
