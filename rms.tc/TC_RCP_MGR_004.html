TC_RCP_MGR_004

Recipe 목록 정렬 (이름 · 수집일 · 버전)

UI Medium
01

Precondition

  1. 서로 다른 이름과 수집일을 가진 Recipe 5개 이상 존재
  2. Recipe Manager 진입
02

Test Steps

  1. 좌측 트리 상단에 정렬 옵션이 있다면 "이름 오름차순" 선택
  2. Recipe 순서 확인
  3. "수집일 내림차순"으로 변경 → 가장 최근 수집된 Recipe가 상단
  4. 같은 Recipe 내 버전 정렬도 확인 (최신 버전이 상단인지 하단인지)
03

Test Data

Recipes (수집일 기준): - 213 (2026-04-13) - RMSRecipeWspinach_gnocci (2026-04-09) - SDC_A1WPP_GZ3_LENS_ADI (2026-04-03)
04

Expected Result

  1. 정렬 옵션 변경 시 트리 순서가 즉시 재정렬됨
  2. 이름 오름차순: 가나다/알파벳 순
  3. 수집일 내림차순: 최근 수집된 Recipe가 상단
  4. 버전 내부 정렬은 일반적으로 최신 버전이 상단(내림차순)을 기본값으로 권장
  5. 정렬 기준이 UI에 명확히 표시됨
05

비고 (QA · 개발 메모)

기능 구현 확인: 정렬 옵션이 명시적 UI 컨트롤로 제공되는지, 컬럼 헤더 클릭 방식인지 확인 필요.

06

제안 사항

UX 개선 정렬 기본값은 수집일 내림차순을 권장드립니다. "최근 변경된 Recipe"를 가장 빨리 볼 수 있어 실무에 가장 유용합니다.

기획 검토 버전 내 정렬은 최신 버전이 상단이 직관적입니다. slide-27에서 v2(2026-04-13)가 v1(2026-04-10) 위에 있는 것과 일치하는지 확인 필요.
