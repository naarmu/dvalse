TC_RCP_XML_006

XML 변환 실패 시 에러 이력 및 알림

Negative High
01

Precondition

  1. 변환 실패를 유도할 XML 파일 준비 (예: 필수 파라미터 누락된 XML)
  2. 서버에 등록 후 장비로 배포
02

Test Steps

  1. 문제 있는 XML 등록 및 배포
  2. 장비에서 변환 실패 발생 대기
  3. RMS UI에서 변환 실패 알림 확인
  4. 이력이 Server Log 또는 Deployment Log에 기록되는지 확인
  5. 실패 사유가 사용자에게 명확히 전달되는지 확인
03

Test Data

XML: invalid_schema.xml (필수 요소 누락) Target: EQ_001
04

Expected Result

  1. 장비에서 변환 실패 발생 시 서버로 실패 알림 전송
  2. RMS UI에 실패 알림 (Toast 또는 알림 배지) 표시
  3. Deployment Log 또는 Server Log에 실패 이력 기록, 원인 포함
  4. 장비의 기존 Recipe가 훼손되지 않음 (원본 유지)
  5. 실패한 XML은 재시도 또는 삭제 가능
05

비고 (QA · 개발 메모)

비동기 알림: 변환은 장비 측에서 일어나므로 서버/UI로의 알림 전달 경로가 정확히 동작해야 함.

06

제안 사항

UX 개선 변환 실패 시 실패 사유를 최대한 상세하게 전달하는 것이 중요합니다. 예: "Line 42: Element 'Size' is required but missing". 단순 "변환 실패"는 사용자가 수정할 수 없습니다.

데이터 무결성 변환 실패 케이스에 대한 대시보드 위젯을 추가하면, 관리자가 실패 경향을 한눈에 파악할 수 있습니다.
