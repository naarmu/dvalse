TC_RCP_XML_001

Custom XML 파일 업로드 및 서버 등록

Functional Critical
01

Precondition

  1. 유효한 Custom XML 파일이 로컬에 준비됨 (예: custom_recipe.xml)
  2. Deploy Custom XML 화면 진입
02

Test Steps

  1. "XML 파일 선택" 또는 드래그 앤 드롭으로 custom_recipe.xml 업로드
  2. 업로드 진행률 표시 확인
  3. 업로드 완료 후 서버에 XML이 등록되었는지 확인 (목록에 표시)
  4. 등록된 XML 선택 시 파일 메타 정보(이름, 크기, 업로더, 일시) 표시 확인
03

Test Data

File: custom_recipe.xml (size: ~50KB) Login: admin_test
04

Expected Result

  1. 업로드 성공 후 서버 XML 목록에 추가됨
  2. 업로드 이력이 기록됨 (업로더, 일시, 파일명, 크기)
  3. 메타 정보 확인 가능
  4. 동일 파일명 재업로드 시 버전 관리 또는 덮어쓰기 정책 적용
05

비고 (QA · 개발 메모)

핵심 기능: Waferless Recipe Creation의 핵심. 장비에서 수집하지 않고 직접 XML로 Recipe를 주입하는 경로.

파일 크기 제한: 서버 수용 최대 크기 정책 확인 필요.

06

제안 사항

보안 업로드된 XML에 서버 측 스키마 검증이 반드시 필요합니다. 악의적 XML(XXE 공격 등)에 대비한 방어 로직을 확인해주세요.

기획 검토 업로드 제한 정책을 명확히 해주세요. 권장: 파일 크기 최대 5MB, 확장자 .xml만 허용.
