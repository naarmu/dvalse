TC_RCP_MGR_010

대용량 Recipe (500+) 목록 로딩 성능

Non-func Medium
01

Precondition

  1. 서버에 500개 이상의 Recipe가 등록된 환경 (사전 세팅)
  2. 각 Recipe당 평균 2~3개 버전 보유
02

Test Steps

  1. Recipe Manager 진입 → 초기 로딩 시간 측정
  2. Recipe 트리 스크롤 부드러움 확인
  3. 랜덤 Recipe 선택 → 상세 로딩 시간 측정
  4. 검색 입력 중 응답성(lag) 확인
  5. CPU · 메모리 사용량 모니터링
03

Test Data

Recipe: 500+ / Version Total: 1,500+
04

Expected Result

  1. 초기 로딩 5초 이내
  2. Recipe 선택 후 상세 표시 1.5초 이내
  3. 스크롤 끊김 없음 (60fps 목표)
  4. 검색 입력 시 응답 지연 300ms 이내
  5. 메모리 500MB 이하, CPU 사용률 평균 30% 이하
05

비고 (QA · 개발 메모)

가상화 여부 확인: TreeListControl에 Virtual Scrolling이 활성화되어야 대용량 대응 가능.

백엔드 페이징: 모든 Recipe를 한 번에 로드하는지, 페이지네이션되는지 API 레벨에서 확인 필요.

06

제안 사항

성능 Recipe 수 1000+ 환경에서도 사용 가능하도록 서버 측 페이지네이션 + 클라이언트 가상 스크롤을 조합하는 것을 권장드립니다.

기획 검토 운영 SLA로 "Recipe 최대 수"를 공식 정의하는 것이 좋습니다. 예: "1 서버당 Recipe 2000개까지 지원".
