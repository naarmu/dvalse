URS_RCP_PRM

파라미터 표시 설정

User Requirement Configuration
01

기능 개요

RMS에 수집된 모든 Recipe 파라미터를 한 화면에서 조회하고, 각 파라미터의 표시 여부를 설정하는 기능이다. 사용자는 이 화면에서 운영상 의미 있는 파라미터만 표시되도록 설정하고, Recipe Manager·Compare Recipes 등 다른 화면은 이 설정에 따라 파라미터를 노출한다.

Recipe별로 수십에서 수백 개의 파라미터가 있을 수 있으나, 일상 운영에서 의미 있는 항목은 그 일부에 한정된다. Manage Recipe Parameters는 그 "의미 있는 항목"의 범위를 정의하는 화면이다.

02

사용자 니즈

주요 사용자

  • 공정 표준화 팀 — 공정 검토에 필요한 핵심 파라미터만 노출하도록 정의
  • RMS 관리자 — 새 파라미터가 수집된 경우 표시 정책 결정

Pain Point (이 기능 도입 전)

  • Recipe Manager나 Compare Recipes에 파라미터가 너무 많이 표시되어 가독성 저하
  • 실제로 거의 변하지 않는 파라미터(장비 ID, 캘리브레이션 값 등)도 항상 노출되어 본질 파악을 방해
  • 파라미터 표시 정책이 사용자별로 달라 공정 검토 시 혼란 발생

기대 효과

  • 운영상 의미 있는 파라미터만 화면에 표시되어 가독성 향상
  • 표시 정책이 전역으로 통일되어 사용자 간 인식 차이 해소
  • 새 파라미터 도입 시 표시 여부를 일관된 기준으로 결정
03

기능 요구사항

■ FR-1. 파라미터 목록 조회

  • FR-1.1 사용자는 RMS에 수집된 모든 Recipe 파라미터를 목록 형태로 조회할 수 있어야 한다.
  • FR-1.2 각 파라미터에 대해 이름, 데이터 타입, 단위, 현재 표시 상태(Show/Hide)가 표시된다.
  • FR-1.3 새로운 Recipe가 수집되어 미등록 파라미터가 발견되면 목록에 자동 추가된다.
  • FR-1.4 파라미터 검색 및 정렬이 가능해야 한다.
  • FR-1.5 파라미터를 카테고리별(예: 측정, 정렬, Tip, Die)로 그룹화하여 표시한다.

■ FR-2. 표시·숨김 설정

  • FR-2.1 사용자는 각 파라미터의 표시 상태를 토글로 변경할 수 있어야 한다.
  • FR-2.2 변경 사항은 명시적 저장 동작으로 서버에 영구 반영된다.
  • FR-2.3 저장된 설정은 사용자가 화면을 재진입해도 유지된다.
  • FR-2.4 변경 이력은 자동 기록된다.

■ FR-3. 다른 화면과의 연동

  • FR-3.1 Hide 설정된 파라미터는 Recipe Manager의 상세 정보 영역에서 표시되지 않는다.
  • FR-3.2 Hide 설정된 파라미터는 Compare Recipes의 비교 테이블에서도 표시되지 않는다.
  • FR-3.3 설정 변경은 다른 화면에 즉시 반영된다.
  • FR-3.4 Compare Recipes 등 연동 화면 상단에 "숨겨진 파라미터 N개" 안내가 표시되어 사용자가 숨김 상태를 인지할 수 있어야 한다.

■ FR-4. 기본값 복원

  • FR-4.1 사용자는 모든 파라미터의 표시 설정을 기본값으로 일괄 초기화할 수 있어야 한다.
  • FR-4.2 기본값은 "전체 표시"이다.
  • FR-4.3 초기화 전 사용자에게 확인을 받는다.
  • FR-4.4 초기화 직전의 설정을 Export하여 백업할 수 있는 옵션을 제공한다.
04

비기능 요구사항

■ NFR-1. 권한

  • NFR-1.1 파라미터 표시 설정 변경은 Vendor, Admin만 수행 가능하다.
  • NFR-1.2 일반 사용자는 현재 설정을 조회할 수 있으나 변경할 수 없다.

■ NFR-2. 일관성

  • NFR-2.1 표시 설정은 전역(모든 사용자 공유)으로 적용된다. 사용자별 개인 설정은 지원하지 않는다.
  • NFR-2.2 모든 파라미터 표시 영역(Recipe Manager, Compare Recipes 등)에서 동일한 설정이 일관되게 반영된다.

■ NFR-3. 성능

  • NFR-3.1 설정 변경 후 다른 화면에 반영되는 시점은 즉시(또는 다음 렌더링 시점) 기준 1초 이내.
05

제약사항 · 정책

  • 정책 파라미터 표시 설정은 전역(global)으로 적용된다. 사용자별 개인 설정은 지원하지 않는다. 공정 검토 시 사용자 간 인식 차이를 방지하기 위함이다.
  • 정책 표시 설정 변경 권한은 Vendor, Admin으로 한정된다.
  • 정책 새로 수집된 미등록 파라미터의 기본 표시 상태는 Show(표시)이다. 관리자가 검토 후 Hide로 변경한다.
  • 정책 기본값 복원의 정의는 "전체 표시"이다.
  • 제약 Hide 설정된 파라미터도 RMS DB에서는 그대로 보존된다. 표시만 제어하며 데이터 자체는 영향받지 않는다.
  • 제약 설정 변경 이력은 RMS Log에 기록된다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 5건:

  • TC_RCP_PRM_001 — 파라미터 전체 목록 표시 (FR-1)
  • TC_RCP_PRM_002 — 파라미터 표시 · 숨김 토글 저장 (FR-2, NFR-1.1)
  • TC_RCP_PRM_003 — 숨김 설정이 Compare Recipes에 반영 (FR-3.2, FR-3.3)
  • TC_RCP_PRM_004 — 숨김 설정이 Recipe Manager 상세에 반영 (FR-3.1, FR-3.3)
  • TC_RCP_PRM_005 — 설정 초기화 (기본값 복원) (FR-4)
