URS_SET_CNN

연결 관리

User Requirement Configuration
01

기능 개요

RMS Client가 사용하는 서버 연결 정보와 장비 Agent 연결 상태를 조회·관리하는 화면이다. 사용자는 현재 연결된 서버와 통신 상태를 확인하고, 필요 시 서버 정보를 변경하거나 연결 테스트를 수행할 수 있다.

Server Setup이 Client 최초 설정 또는 서버 이전 시 사용된다면, Connection은 운영 중에 연결 상태를 점검하고 진단하는 화면이다.

02

사용자 니즈

주요 사용자

  • RMS 관리자 — 서버 연결 상태 점검
  • 장비 엔지니어 — Agent 연결 문제 진단

Pain Point (RMS 도입 전)

  • 연결이 끊어졌을 때 원인을 진단할 도구가 없음
  • Agent가 정상 동작하는지 별도 도구로 확인해야 함
  • 서버 정보를 바꾸려면 설정 파일을 직접 수정

기대 효과

  • 현재 연결 상태를 단일 화면에서 확인
  • UI에서 직접 연결 테스트 수행
  • Agent별 연결 상태의 실시간 가시화
03

기능 요구사항

■ FR-1. 연결 정보 조회

  • FR-1.1 화면 진입 시 현재 사용 중인 서버 IP, 포트, TLS 사용 여부, 인증서 정보가 표시된다.
  • FR-1.2 마지막 통신 시각, 통신 지연 시간(ping)이 표시된다.
  • FR-1.3 등록된 모든 장비의 Agent 연결 상태(Online/Offline)가 목록 형태로 표시된다.

■ FR-2. 연결 정보 변경

  • FR-2.1 사용자는 서버 IP, 포트 등 연결 정보를 변경할 수 있어야 한다.
  • FR-2.2 변경 후 저장 시 Client는 즉시 새 정보로 재연결을 시도한다.
  • FR-2.3 재연결이 실패한 경우 이전 정보로 자동 롤백되거나, 사용자에게 처리 방식 선택을 받는다.

■ FR-3. 연결 정보 검증

  • FR-3.1 IP 형식, 포트 범위 검증을 입력 단계에서 수행한다.
  • FR-3.2 검증 실패 시 명확한 에러 메시지를 표시한다.
  • FR-3.3 연결 실패 시 사유(타임아웃 / 거부 / 인증서 오류 등)별로 구분된 메시지를 표시한다.

■ FR-4. 연결 테스트

  • FR-4.1 사용자는 "연결 테스트" 버튼으로 즉시 서버 도달성을 확인할 수 있어야 한다.
  • FR-4.2 테스트 결과(성공 / 실패, 응답 시간)가 화면에 표시된다.
  • FR-4.3 사용자는 특정 장비를 선택하여 해당 Agent와의 연결 테스트도 수행할 수 있다.

■ FR-5. Agent 실시간 표시

  • FR-5.1 각 Agent의 연결 상태가 실시간으로 갱신된다.
  • FR-5.2 연결 상태 변경 시 화면에 즉시 반영된다.
  • FR-5.3 장기 Offline Agent는 별도 강조 표시한다.
04

비기능 요구사항

■ NFR-1. 권한

  • NFR-1.1 연결 정보 조회는 모든 사용자(All Roles)가 가능하다.
  • NFR-1.2 연결 정보 변경은 Vendor, Admin만 수행 가능하다.
  • NFR-1.3 연결 테스트는 Vendor, Admin, Engineer가 수행 가능하다.

■ NFR-2. 보안

  • NFR-2.1 Client–서버 통신은 TLS로 암호화된 채널 위에서 수행되어야 한다.
  • NFR-2.2 인증서 정보 표시 시 민감한 비밀키는 노출되지 않는다.

■ NFR-3. 사용성

  • NFR-3.1 연결 실패 메시지는 사용자가 다음 행동을 결정할 수 있도록 명확해야 한다.
  • NFR-3.2 연결 테스트는 5초 이내에 결과를 표시해야 한다.
05

제약사항 · 정책

  • 정책 연결 정보 변경 후 재연결 실패 시 이전 정보로 자동 롤백하거나 사용자에게 처리 방식을 선택받는다. 잘못된 변경으로 시스템이 마비되는 것을 방지한다.
  • 정책 연결 실패 메시지는 사유별로 구분한다(타임아웃 / 거부 / 인증서 오류 등).
  • 정책 Agent 장기 Offline은 별도 강조 표시한다. 비정상 장비를 빠르게 발견하기 위함이다.
  • 제약 연결 정보 변경 권한은 Vendor, Admin로 한정된다.
  • 제약 Client–서버 통신은 TLS 암호화 채널 위에서만 허용된다.
  • 제약 연결 테스트 응답은 5초 이내에 표시되어야 한다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 5건:

  • TC_SET_CNN_001 — 서버 · 네트워크 연결 정보 조회 (FR-1)
  • TC_SET_CNN_002 — 연결 정보 변경 및 재연결 (FR-2)
  • TC_SET_CNN_003 — 잘못된 연결 정보 입력 시 에러 처리 (FR-3)
  • TC_SET_CNN_004 — 연결 테스트 (Ping · Connect Test) 기능 (FR-4)
  • TC_SET_CNN_005 — Agent 연결 상태 실시간 표시 (FR-5)
