TC_RCP_DEP_005

배포 중 장비 연결 끊김 시 에러 처리

Negative Critical
01

Precondition

  1. Online 장비 EQ_001에 Recipe 배포 중
  2. 배포 진행률이 30~70% 사이인 상태
02

Test Steps

  1. 배포 진행 중 EQ_001의 네트워크 연결을 강제 해제 (케이블 분리 또는 Agent 종료)
  2. UI에서 에러 처리 동작 확인
  3. Deployment Log에 실패 기록 확인
  4. EQ_001 재연결 후 장비 상태 확인 — Recipe가 중간 상태로 남지 않았는지
03

Test Data

Trigger: 네트워크 해제 (Deploy 진행률 50% 시점)
04

Expected Result

  1. 배포 실패 인식 및 적절한 에러 메시지 표시 ("장비와의 연결이 끊어졌습니다")
  2. Deployment Log에 실패 이력 기록 (실패 원인 포함)
  3. 장비가 부분 전송된 Recipe로 불안정한 상태가 되지 않도록 Rollback 또는 Retry 메커니즘 동작
  4. 재연결 시 자동 복구 또는 사용자에게 재시도 유도
05

비고 (QA · 개발 메모)

무결성 중요: 반도체 공정 환경에서 "파일이 부분적으로 쓰여진 상태"는 장비 오작동의 원인이 될 수 있음. Atomic Write 방식(임시 파일로 쓴 후 rename) 적용 여부 확인 필수.

06

제안 사항

데이터 무결성 Agent 측에서 임시 파일로 먼저 받은 후 전체 수신 완료 시 최종 rename하는 방식이 반드시 적용되어야 합니다. 전송 중 끊김이 있어도 장비의 기존 Recipe가 손상되지 않도록.

UX 개선 실패한 배포를 "재시도" 버튼 한 번으로 재실행할 수 있도록 Deployment Log에서 바로 재시도 기능 제공 권장.
