TC_RCP_CMP_002

2개 Recipe 선택 후 파라미터 비교 표시

Functional Critical
01

Precondition

  1. 비교할 Recipe 2개가 서로 다른 파라미터 값을 가지고 있음
  2. Compare Recipes 화면 진입
02

Test Steps

  1. Recipe A와 Recipe B를 각각 선택
  2. 비교 결과 테이블에서 파라미터별 값이 나란히 표시되는지 확인
  3. 동일한 파라미터는 일반 표시, 다른 값은 하이라이트 확인
  4. Recipe A에만 있고 B에는 없는 파라미터의 표시 방식 확인
03

Test Data

Recipe A: Size=300, Die X=20.0, Tip ID=NCHR Recipe B: Size=200, Die X=20.0, Tip ID=AR5T
04

Expected Result

  1. 파라미터 목록이 테이블 행으로, 각 Recipe가 컬럼으로 표시
  2. 값이 다른 행은 배경색 하이라이트 (예: 연분홍)
  3. 값이 같은 행은 플레인 표시
  4. 한쪽에만 있는 파라미터는 다른 쪽에 "—" 또는 빈 값 + 별도 표시
  5. slide-29(Compare Recipe) UI와 일치하는 레이아웃
05

비고 (QA · 개발 메모)

slide-29 확인: 현재 화면에서 중앙에 "Recipe Contents"가 표시되고 우측에 "Equipment in Deploy"가 있는 구조임. 실제 비교 결과 표시 방식 재확인 필요.

파라미터 개수: 100개 이상 파라미터일 때 테이블이 길어질 수 있어 필터/검색 기능이 있으면 좋음.

06

제안 사항

UX 개선 비교 결과에서 "차이만 보기" 토글 버튼을 제공하면 수많은 파라미터 중 변경된 것만 빠르게 확인 가능합니다.

정보 구조 하이라이트 색상을 의미에 따라 3단계로 구분하는 것도 고려: ① 값 다름(노랑), ② 한쪽에만 존재(파랑), ③ 단위 차이(주황).
