TC_RCP_VER_006

특정 버전 삭제 및 제약조건

Functional High
01

Precondition

  1. Recipe "213"이 v1, v2, v3 세 버전 보유
  2. v2는 Master로 지정됨
  3. v1, v3은 Master 아님
02

Test Steps

  1. v1 선택 후 삭제 시도 → 삭제 성공 확인
  2. v2(Master) 선택 후 삭제 시도 → 차단되거나 경고 확인
  3. 남은 v3만 있는 상태에서 v3 삭제 시도 → 마지막 버전 삭제 정책 확인
03

Test Data

Recipe 213: v1, v2 (Master), v3
04

Expected Result

  1. 일반 버전(v1, v3) 삭제 가능
  2. Master 버전(v2)은 삭제 차단 또는 "먼저 다른 버전을 Master로 지정하세요" 안내
  3. 마지막 남은 버전 삭제 시 Recipe 자체 삭제 여부를 사용자 확인
  4. 삭제 이력 기록
05

비고 (QA · 개발 메모)

정책 결정 필요: 마지막 버전 삭제 시 Recipe 자체를 삭제할지, 차단할지.

06

제안 사항

정책 확정 Master 버전 삭제 정책을 확정해주세요. 권장: "먼저 Master를 다른 버전으로 옮겨라"로 차단. 이는 실수 방지에 효과적.

데이터 무결성 버전도 소프트 삭제(Soft Delete)를 권장드립니다. 삭제 후 30일 복원 가능하도록.
