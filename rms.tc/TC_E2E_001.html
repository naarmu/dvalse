TC_E2E_001

장비 등록 → Recipe 자동 수집 → Master 지정 → 다른 장비로 배포 → 이력 확인

Functional Critical
01

Precondition

  1. RMS Server 정상 동작
  2. Agent 설치된 장비 PC 2대 이상 준비 (EQ_001, EQ_002)
  3. EQ_001에는 기존 Recipe 파일이 존재, EQ_002는 동일 장비 모델이되 Recipe 없음
02

Test Steps

  1. Admin 계정 로그인 (TC_SYS_LGN_001 연계)
  2. Equipment Registration에서 EQ_001 등록 (TC_EQP_REG_001 연계)
  3. EQ_001 등록 직후 Recipe 자동 수집 동작 확인 (TC_CLT_RCP_001 연계)
  4. Pending Recipes 팝업에서 수집된 Recipe 서버 등록 (TC_SYS_PND_003 연계)
  5. Recipe Manager에서 등록된 Recipe 확인 (TC_RCP_MGR_001, 002 연계)
  6. Manage Recipe Versions에서 해당 Recipe의 v1을 Master로 지정 (TC_RCP_VER_002 연계)
  7. EQ_002를 Equipment Registration에서 등록
  8. Deploy Recipes에서 해당 Recipe Master 버전을 EQ_002로 배포 (TC_RCP_DEP_002 연계)
  9. 배포 완료 후 Deployment Log 확인 (TC_HST_DEP_001 연계)
  10. Recipe History에서 전체 이력(등록 → Master 지정 → 배포) 시계열 확인
03

Test Data

Start Point: 장비 2대 + 서버 빈 DB End Point: 두 장비에 동일 Recipe Master 배포 완료 + 이력 완성
04

Expected Result

  1. 모든 단계가 순차적으로 정상 완료
  2. 각 단계의 이력이 전부 기록되어 시계열 추적 가능
  3. 최종 상태: 두 장비 모두 동일 Recipe 배포, Master 지정 명확
  4. 단계별 UI 전환이 매끄럽고 데이터 정합성 유지
05

비고 (QA · 개발 메모)

핵심 플로우: RMS의 가장 전형적인 운영 시나리오. 이 플로우가 매끄럽게 동작해야 제품 가치가 입증됨.

소요 시간: 실제 수행 시 15분 내외. 자동화 테스트 대상 1순위.

06

제안 사항

QA 자동화 이 시나리오는 E2E 자동화 테스트 우선순위 1순위로 권장드립니다. UI 변경이 있을 때마다 반드시 회귀 검증이 필요한 플로우입니다.

UX 개선 단계별로 다음 단계로의 "다음 단계로 이동" 버튼이 있으면 신규 사용자 온보딩에 큰 도움이 됩니다. 예: 장비 등록 완료 화면에서 "Pending Recipes 확인하러 가기" 버튼.
