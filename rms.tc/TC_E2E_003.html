TC_E2E_003

Custom XML 업로드 → 장비 배포 → .psr 자동 변환 → Recipe Manager 조회 → 재배포

Functional Critical
01

Precondition

  1. 유효한 Custom XML 파일 준비
  2. Online 장비 EQ_001, EQ_002
  3. Engineer 이상 권한
02

Test Steps

  1. Deploy Custom XML 화면에서 XML 업로드 (TC_RCP_XML_001 연계)
  2. 업로드된 XML을 EQ_001로 배포 (TC_RCP_XML_003 연계)
  3. EQ_001에서 XML → .psr 자동 변환 완료 대기 (TC_RCP_XML_004 연계)
  4. 변환된 .psr이 Agent에 의해 서버로 수집되는지 확인
  5. Recipe Manager에서 신규 Recipe로 조회 가능 확인 (TC_RCP_XML_005 연계)
  6. 해당 Recipe의 파라미터 값이 XML 정의와 일치하는지 검증
  7. 새로 생성된 Recipe를 EQ_002에도 배포 (일반 Recipe처럼 동작 확인)
  8. EQ_002에도 동일한 .psr이 저장되었는지 확인
  9. Deployment Log에서 "Custom XML" 배포와 "Recipe" 배포가 구분되어 기록되는지 확인
03

Test Data

Custom XML: wafer_recipe.xml (50KB, 20 parameters) End Goal: 2대 장비에 동일 Recipe 배포 완료
04

Expected Result

  1. XML → PSR 변환이 Atomic하게 수행 (중간 상태 파일 없음)
  2. 변환된 Recipe가 일반 Recipe와 동일하게 기능
  3. Deployment Log에 Type 구분 기록
  4. 전체 플로우 총 15분 이내
  5. Custom XML 출처 Recipe에 시각적 구분 표시
05

비고 (QA · 개발 메모)

Phase 2 핵심 가치: Waferless Recipe Creation 기능의 End-to-End 검증. 제품 차별점.

변환 시점: 장비 측 변환 완료 시간(수 초~수십 초 추정)이 UX 체감에 큰 영향.

06

제안 사항

QA 자동화 Phase 2 신규 기능 중 가장 복잡한 플로우이므로 자동화 우선순위 1순위입니다. 회귀 방지 필수.

UX 개선 XML 업로드 후 변환 완료까지 진행 상태를 시각화하여 사용자가 어느 단계에 있는지 파악 가능하게 해주세요. "① 업로드 → ② 배포 → ③ 변환 중 → ④ 수집 → ⑤ 완료" 단계 표시.
