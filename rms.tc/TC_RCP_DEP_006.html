TC_RCP_DEP_006

배포 완료 후 Deployment Log 기록

Functional Critical
01

Precondition

  1. Deploy Recipes에서 배포가 정상 완료된 직후
02

Test Steps

  1. 배포 완료 확인
  2. History → Deployment Log로 이동
  3. 방금 수행한 배포 이력이 최상단에 있는지 확인
  4. 기록된 항목(배포자, 시각, Recipe, 버전, 장비, 결과) 정확성 확인
03

Test Data

Deployed: Recipe 213 v2 → EQ_001, EQ_002 By: admin_test Time: (방금)
04

Expected Result

  1. Deployment Log에 방금 배포 이력이 기록됨
  2. 필수 필드 모두 채워짐: 배포자 계정, 일시, Recipe 이름/버전, 대상 장비, 결과(성공/실패)
  3. 여러 장비 배포 시 장비별 각각 레코드 또는 하나의 레코드에 장비 목록 포함 — 정책에 따름
  4. 실패한 항목은 실패 사유 함께 기록
05

비고 (QA · 개발 메모)

감사(Audit) 관점: 반도체 공정 환경에서 Deployment Log는 SEMI 표준 등 외부 감사 대상 가능성 있음. 기록 완전성·불변성 중요.

06

제안 사항

데이터 무결성 Deployment Log는 수정·삭제 불가(append-only)여야 합니다. 기록된 이력은 어떤 사용자도 변경할 수 없어야 하며, 권한 체계에서도 명확히 차단되어야 합니다.

기획 검토 배포 이력의 보존 기간(Retention Policy) 정의가 필요합니다. 권장: 최소 3년. 반도체 업계 감사 요구사항 기준.
