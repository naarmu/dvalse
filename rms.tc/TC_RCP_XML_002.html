TC_RCP_XML_002

잘못된 형식 XML 파일 거부

Negative Critical
01

Precondition

  1. 테스트용 잘못된 형식의 파일 준비
02

Test Steps

  1. 확장자가 .xml이 아닌 파일 업로드 시도 (예: custom.txt)
  2. 에러 메시지 확인
  3. 확장자는 .xml이지만 내용이 XML 문법 오류인 파일 업로드
  4. 에러 메시지 확인
  5. XML 문법은 맞지만 Recipe 스키마를 따르지 않는 파일 업로드
  6. 에러 메시지 확인
03

Test Data

Case 1: custom.txt (확장자 불일치) Case 2: broken.xml (... 닫히지 않음) Case 3: unrelated.xml (XML이지만 Recipe 스키마 아님)
04

Expected Result

  1. Case 1: 확장자 체크로 즉시 차단, "XML 파일만 업로드 가능합니다" 메시지
  2. Case 2: 서버 측 XML 파싱 실패, "XML 형식이 올바르지 않습니다" 메시지
  3. Case 3: 스키마 검증 실패, "Recipe XML 스키마에 맞지 않습니다" 메시지
  4. 모든 케이스에서 서버에 파일이 저장되지 않음
05

비고 (QA · 개발 메모)

검증 단계: 클라이언트 확장자 검증 + 서버 형식 검증 + 서버 스키마 검증, 3단계 방어 필요.

06

제안 사항

보안 XML 파싱 시 외부 엔티티 참조(XXE)를 비활성화하는 파서 설정이 필수입니다. 보안 취약점의 대표 사례.

UX 개선 검증 실패 메시지에 "예시 파일 다운로드" 링크를 함께 제공하면, 사용자가 올바른 형식을 빠르게 파악할 수 있습니다.
