URS_EQP_REG

장비 등록

User Requirement Core Function
01

기능 개요

새 장비를 RMS에 등록하는 화면이다. 사용자는 장비의 식별 정보(ID, IP, 모델 등)를 입력하여 RMS가 해당 장비를 인식하고, 장비에 설치된 Agent와 통신할 수 있도록 한다.

장비 등록은 RMS 운영의 시작 단계이다. 등록되지 않은 장비는 Recipe·Method 수집·배포 대상이 될 수 없다.

02

사용자 니즈

주요 사용자

  • RMS 관리자 — 신규 장비 도입 시 RMS에 추가
  • 장비 엔지니어 — 장비 교체·이전 시 RMS 등록 정보 변경

Pain Point (RMS 도입 전)

  • 장비 식별 정보가 개별 시스템에 분산되어 통합 관리 어려움
  • 장비 교체 시 식별 정보 갱신 누락으로 운영 사고 발생

기대 효과

  • 모든 장비 식별 정보의 중앙 관리
  • 등록 단계의 입력 검증으로 잘못된 데이터 차단
  • 등록 후 Agent 자동 연결로 운영 즉시 가능
03

기능 요구사항

■ FR-1. 신규 장비 등록

  • FR-1.1 사용자는 신규 장비의 식별 정보를 입력하여 등록할 수 있어야 한다.
  • FR-1.2 입력 필드는 장비 ID(필수), 장비명(필수), IP(필수), 모델(필수), 소속 그룹(선택), 설명(선택)을 포함한다.
  • FR-1.3 등록 성공 시 Equipment Manager 목록에 해당 장비가 표시된다.
  • FR-1.4 등록 성공 시 사용자에게 다음 단계(Agent 설치 안내) 정보를 제공한다.

■ FR-2. 입력값 검증

  • FR-2.1 장비 ID는 RMS 내에서 유일해야 한다. 중복 ID 입력 시 등록을 차단한다.
  • FR-2.2 IP 주소는 IPv4 형식 검증을 거쳐야 한다. 형식 오류 시 등록을 차단한다.
  • FR-2.3 필수 입력 필드가 비어 있는 경우 등록 버튼이 비활성 또는 에러 표시된다.
  • FR-2.4 검증 실패 시 어느 필드의 어떤 값이 문제인지 명확한 에러 메시지를 표시한다.

■ FR-3. 등록 후 Agent 연결

  • FR-3.1 장비 등록 직후의 연결 상태는 Offline(Agent 미연결)으로 표시된다.
  • FR-3.2 해당 장비에 Agent가 설치되고 RMS 서버 정보가 설정되면, Agent가 자동으로 서버에 접속하여 Online 상태로 전환된다.
  • FR-3.3 Agent 자동 연결 후 해당 장비의 Recipe·Method가 자동 수집되어 Pending Recipes 목록에 표시된다.

■ FR-4. 등록 취소 · 삭제

  • FR-4.1 사용자는 등록 화면에서 입력 도중 취소할 수 있어야 한다. 취소 시 입력 내용은 저장되지 않는다.
  • FR-4.2 이미 등록된 장비를 삭제할 수 있어야 한다. 삭제 시 사용자에게 확인을 받는다.
  • FR-4.3 장비 삭제 시 해당 장비의 등록 정보가 RMS에서 제거되며, 관련 이력은 보존된다.
  • FR-4.4 장비 삭제 이력은 RMS Log에 자동 기록된다.
04

비기능 요구사항

■ NFR-1. 권한

  • NFR-1.1 장비 등록·수정·삭제는 Vendor, Admin만 수행 가능하다.
  • NFR-1.2 권한 검증은 UI 제약과 서버 API 이중 방어로 수행되어야 한다.

■ NFR-2. 데이터 무결성

  • NFR-2.1 등록·수정·삭제 이력은 RMS Log에 자동 기록되며 수정·삭제할 수 없다.
  • NFR-2.2 장비 삭제 시 해당 장비가 남긴 운영 이력(배포 이력 등)은 절대 함께 삭제되지 않는다.

■ NFR-3. 가시성

  • NFR-3.1 입력 필드의 검증 결과는 사용자가 입력하는 도중에 실시간으로 표시되어야 한다.
05

제약사항 · 정책

  • 정책 장비 ID는 RMS 내에서 유일하다. 중복 등록은 차단한다.
  • 정책 등록 직후의 장비는 Offline 상태이며, Agent 연결 시 Online으로 자동 전환된다.
  • 정책 장비 삭제 시 해당 장비가 남긴 이력(배포 이력, History 등)은 보존된다. 이력 무결성을 위함이다.
  • 정책 입력값 검증은 클라이언트 단계와 서버 단계 양쪽에서 수행한다.
  • 제약 등록·수정·삭제 권한은 Vendor, Admin로 한정된다.
  • 제약 한 장비는 단일 그룹에만 소속된다.
06

  • TC_EQP_REG_003 — 잘못된 IP 형식 입력 시 검증 (FR-2.2, NFR-3.1)
  • TC_EQP_REG_004 — 필수 입력 누락 시 등록 차단 (FR-2.3, FR-2.4)
  • TC_EQP_REG_005 — 등록 후 Agent 미설치 장비의 미연결 상태 (FR-3.1)
  • TC_EQP_REG_006 — 등록 후 Agent 설치 장비의 자동 연결 (FR-3.2, FR-3.3)
  • TC_EQP_REG_007 — 장비 등록 취소 동작 (FR-4.1)
