TC_SYS_PND_002

Pending 목록의 장비명 · Recipe명 · 변경유형 표시

UI High
01

Precondition

  1. Pending 항목이 여러 개 존재
02

Test Steps

  1. Pending Recipes 팝업 열기
  2. 목록 컬럼 확인 (장비명, Recipe명, 변경유형, 감지 시각 등)
  3. 장비별 그룹핑 여부 확인
03

Test Data

Pending: EQ_001 / Recipe_A / 신규 / 2026-04-24 10:00 EQ_001 / Recipe_B / 수정 / 2026-04-24 10:05 EQ_002 / Recipe_C / 신규 / 2026-04-24 10:10
04

Expected Result

  1. 각 Pending 항목에 장비명, Recipe명, 변경유형(신규/수정/삭제), 감지 시각 표시
  2. 장비별 또는 시간순 정렬
  3. 변경유형 아이콘 또는 색상 구분
05

비고 (QA · 개발 메모)

변경유형 정의: 신규(Created), 수정(Modified), 삭제(Deleted) 3종. Agent가 파일 변화 감지로 판단.

06

제안 사항

정보 구조 변경유형에 색상 구분을 적용하면 한눈에 파악이 쉽습니다. 신규=초록, 수정=주황, 삭제=빨강 권장.
