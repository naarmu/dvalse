URS_DSH_RMS

RMS 시스템 대시보드

User Requirement Overview
01

기능 개요

RMS 시스템 전체의 운영 현황을 요약하여 한 화면에서 보여주는 대시보드이다. 사용자는 이 화면을 통해 RMS 서버의 상태, 연결된 장비 수, 등록된 Recipe 수 등 핵심 지표를 일별로 확인한다.

RMS Dashboard는 일상 점검의 출발점이며, 평소와 다른 점이 있는지 빠르게 인지하기 위한 화면이다. 상세 작업이 필요한 경우 각 위젯에서 해당 영역의 상세 화면으로 이동한다.

02

사용자 니즈

주요 사용자

  • RMS 관리자 — 시스템 전반의 일별 점검
  • 장비 엔지니어 — 운영 시작 전 환경 확인
  • 공정 표준화 팀 — 자산 현황 조감

Pain Point (RMS 도입 전)

  • 시스템 상태를 확인하려면 여러 화면을 순회해야 함
  • 이상 징후가 있어도 화면 진입 전에는 인지할 수 없음

기대 효과

  • 로그인 직후 시스템 상태를 즉시 파악
  • 이상 징후를 위젯의 시각 표현으로 빠르게 인지
  • 위젯 클릭으로 상세 화면 직접 이동
03

기능 요구사항

■ FR-1. 서버 상태 위젯

  • FR-1.1 Dashboard 진입 시 RMS 서버의 가동 상태(정상/경고/오프라인)를 표시하는 위젯이 표시되어야 한다.
  • FR-1.2 서버 상태에 따라 위젯의 색상과 아이콘이 시각적으로 구분된다.
  • FR-1.3 서버가 비정상 상태인 경우 위젯에 사유와 마지막 정상 통신 시각이 함께 표시된다.

■ FR-2. 장비 집계 위젯

  • FR-2.1 등록된 전체 장비 수, Online 장비 수, Offline 장비 수가 위젯에 표시된다.
  • FR-2.2 Offline 장비 수가 임계값 이상인 경우 시각 강조 표시한다.
  • FR-2.3 위젯의 수치를 클릭하면 Equipment Manager로 이동하며 해당 상태로 자동 필터링된다.

■ FR-3. Recipe · Method 집계 위젯

  • FR-3.1 등록된 전체 Recipe 수와 Method 수가 위젯에 표시된다.
  • FR-3.2 Master 미지정 Recipe 수도 함께 표시되어, 관리자가 정리 필요성을 인지할 수 있다.

■ FR-4. 실시간 반영

  • FR-4.1 Dashboard의 모든 위젯은 일정 주기(30초 권장)로 자동 갱신된다.
  • FR-4.2 사용자는 수동 새로고침 버튼으로 즉시 갱신할 수 있다.
  • FR-4.3 마지막 갱신 시각이 화면 어딘가에 표시되어 사용자가 데이터 신선도를 판단할 수 있다.

■ FR-5. Empty 상태 처리

  • FR-5.1 등록된 장비 또는 Recipe가 0개인 경우 Empty State 안내와 다음 액션 유도가 표시된다.
  • FR-5.2 빈 위젯은 시각적으로 깨지지 않고 정돈된 상태로 표시되어야 한다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 Dashboard 전체 로딩은 5초 이내 완료되어야 한다.
  • NFR-1.2 위젯별 비동기 로딩으로 일부 위젯이 늦더라도 다른 위젯이 차단되지 않아야 한다.
  • NFR-1.3 집계 데이터는 사전 계산된 형태로 제공되어 실시간 집계 부하를 최소화한다.

■ NFR-2. 권한

  • NFR-2.1 Dashboard는 모든 사용자(All Roles)가 조회 가능하다.

■ NFR-3. 가시성

  • NFR-3.1 상태 표시는 색상만으로 의미를 전달하지 않는다. 색상 + 아이콘 또는 색상 + 텍스트 라벨 병행을 원칙으로 한다.
05

제약사항 · 정책

  • 정책 Dashboard 자동 갱신 주기는 30초이다. 너무 짧으면 서버 부하가 늘고, 너무 길면 운영 가시성이 떨어진다.
  • 정책 마지막 갱신 시각을 화면에 표시한다. 사용자가 데이터 신선도를 판단할 수 있어야 한다.
  • 정책 집계 데이터는 사전 계산된 형태로 제공한다. 실시간 집계는 부하가 크기 때문이다.
  • 정책 위젯 클릭 시 해당 영역의 상세 화면으로 자동 이동하며, 가능한 경우 컨텍스트(필터 등)도 자동 적용된다.
  • 제약 Dashboard 위젯 데이터는 사용자별 개인화 없이 시스템 전체 기준으로 표시된다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 5건:

  • TC_DSH_RMS_001 — RMS Dashboard 진입 시 서버 상태 위젯 표시 (FR-1)
  • TC_DSH_RMS_002 — 연결된 장비 수 집계 정확성 (FR-2)
  • TC_DSH_RMS_003 — 등록된 Recipe 수 집계 정확성 (FR-3)
  • TC_DSH_RMS_004 — 서버 상태 변경 시 Dashboard 실시간 반영 (FR-4)
  • TC_DSH_RMS_005 — Empty 상태 (장비 · Recipe 0개) 표시 (FR-5)
