TC_E2E_002

장비측 Recipe 변경 → Pending 감지 → 서버 등록 → 버전 비교 → 이전 버전 롤백 배포

Functional Critical
01

Precondition

  1. EQ_001에 Recipe X v1이 배포된 상태 (Master)
  2. Agent 정상 동작
  3. Engineer 이상 권한 계정 로그인
02

Test Steps

  1. 장비 PC의 Recipe X 파일을 직접 편집 (파라미터 값 변경)
  2. Agent가 변경 감지하여 Pending Recipes에 자동 보고 (TC_CLT_RCP_001 연계)
  3. Pending Recipes에서 변경된 Recipe를 서버에 v2로 등록
  4. Recipe Manager에서 Recipe X가 v1, v2 두 버전 보유 확인
  5. Compare Recipes로 v1, v2 파라미터 비교 (TC_RCP_CMP_005 연계)
  6. 변경 결과 검토 후 문제 판단 → 롤백 결정
  7. Manage Recipe Versions에서 Master를 v2 → v1로 되돌림 (TC_RCP_VER_002 연계)
  8. Deploy Recipes에서 v1(현재 Master)을 EQ_001로 재배포
  9. 장비 PC에서 Recipe가 v1 내용으로 복구되었는지 확인
  10. Recipe History에서 변경 → Master 변경 → 롤백 배포의 전체 흐름 확인
03

Test Data

Recipe X v1 (Die X=20.0, Tip ID=NCHR) → 장비에서 직접 수정 → v2 (Die X=25.0) 롤백 후: v1 재배포
04

Expected Result

  1. 장비측 변경이 실시간(또는 주기 내) 감지
  2. 버전 비교 결과가 정확 (변경 파라미터 하이라이트)
  3. Master 재지정 정상 동작
  4. 롤백 배포 완료 후 장비 Recipe가 이전 내용으로 완전 복구
  5. 전체 이력이 시계열로 추적 가능
05

비고 (QA · 개발 메모)

공정 안전: 반도체 공정에서 Recipe 변경 후 문제가 발견되면 빠른 롤백이 필수. 이 플로우의 속도가 곧 공정 다운타임을 결정함.

06

제안 사항

UX 개선 "이전 버전으로 빠른 롤백" 원클릭 버튼을 Recipe Manager에서 제공하는 것을 강하게 권장드립니다. 롤백은 긴급 상황에서 자주 쓰이는 기능입니다.

데이터 무결성 롤백 배포도 일반 배포와 동일하게 Deployment Log에 기록되되, "Rollback" 표시를 추가하면 운영 분석에 유용합니다.
