URS_HST_SVR

서버 시스템 로그

User Requirement Diagnostics
01

기능 개요

RMS 서버에서 발생한 시스템 이벤트(서비스 시작·종료, 설정 변경, 오류 발생, Agent 연결 변화 등)의 시계열 로그를 조회하는 화면이다. RMS Log가 사용자 행위를 다룬다면, Server Log는 서버 자체의 운영 이벤트를 다룬다.

Server Log는 시스템 장애 진단과 운영 안정성 모니터링을 위한 화면이다. 서버 재기동 시점, 설정 변경 시점, 비정상 종료 등의 시스템 이벤트를 추적한다.

02

사용자 니즈

주요 사용자

  • Vendor — 시스템 운영 안정성 검증
  • RMS 관리자 — 시스템 장애 발생 시 원인 분석

Pain Point (RMS 도입 전)

  • 서버 시스템 이벤트가 별도 로그 파일로만 존재
  • 장애 발생 시 SSH 접속 등 추가 절차로만 로그 확인 가능
  • 로그 분석 도구가 없어 패턴 파악 어려움

기대 효과

  • 서버 시스템 이벤트의 UI 기반 조회
  • 장애 시점 추적의 즉시성
  • 필터·검색을 통한 패턴 분석 가능
03

기능 요구사항

■ FR-1. 서버 이벤트 이력 조회

  • FR-1.1 화면 진입 시 서버 시스템 이벤트가 시계열(최신 우선)로 표시된다.
  • FR-1.2 각 항목에는 발생 시각, 이벤트 레벨(Info / Warning / Error / Critical), 이벤트 유형, 상세 내용이 표시된다.
  • FR-1.3 이벤트 유형은 Service Start / Service Stop / Config Change / Error / Agent Connect / Agent Disconnect 등을 포함한다.

■ FR-2. 이벤트 상세 표시

  • FR-2.1 사용자는 항목을 선택하여 상세 정보(스택 트레이스, 컨텍스트 등)를 확인할 수 있어야 한다.
  • FR-2.2 Error·Critical 레벨 이벤트는 별도 색상·아이콘으로 강조 표시한다.

■ FR-3. 필터 · 검색

  • FR-3.1 사용자는 이벤트 레벨, 유형, 시간 범위로 필터링할 수 있어야 한다.
  • FR-3.2 사용자는 키워드로 검색할 수 있어야 한다.
  • FR-3.3 다중 필터를 동시 적용할 수 있다(AND 조건).

■ FR-4. Export

  • FR-4.1 사용자는 현재 필터링 결과를 CSV 또는 Excel로 Export할 수 있어야 한다.
  • FR-4.2 Export 시 상세 정보(스택 트레이스 등)도 포함된다.

■ FR-5. 데이터 보존

  • FR-5.1 서버 로그는 어떤 사용자도 수정·삭제할 수 없다(append-only).
  • FR-5.2 이력 보존 기간은 최소 1년이다(상세 로그의 보존 비용을 고려한 기준).
  • FR-5.3 Critical·Error 레벨 이벤트는 최소 3년 보존한다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 화면 진입 후 초기 데이터 표시까지 5초 이내.
  • NFR-1.2 필터 적용 응답은 3초 이내.

■ NFR-2. 권한

  • NFR-2.1 Server Log 조회는 Vendor, Admin만 가능하다.
  • NFR-2.2 Export는 Vendor, Admin이 가능하다.
  • NFR-2.3 일반 사용자(Engineer, Operator)는 Server Log에 접근할 수 없다.

■ NFR-3. 보안

  • NFR-3.1 로그 상세 정보에 비밀번호·토큰 등 민감 정보가 노출되지 않도록 자동 마스킹한다.
05

제약사항 · 정책

  • 정책 서버 로그는 append-only로 관리된다.
  • 정책 보존 기간은 일반 로그 1년, Critical·Error 레벨 3년으로 차등 적용한다. 보존 비용과 감사 요구사항의 균형을 위함이다.
  • 정책 Error·Critical 이벤트는 시각적으로 강조 표시한다.
  • 정책 로그 상세 정보의 민감 데이터(비밀번호·토큰 등)는 자동 마스킹된다.
  • 제약 Server Log 접근 권한은 Vendor, Admin로 한정된다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 4건:

  • TC_HST_SVR_001 — 서버 변경 이력 조회 (FR-1)
  • TC_HST_SVR_002 — 서버 이벤트 상세 표시 (FR-2)
  • TC_HST_SVR_003 — 필터 · 검색 기능 (FR-3)
  • TC_HST_SVR_004 — Log Export 기능 (FR-4)
