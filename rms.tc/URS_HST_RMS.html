URS_HST_RMS

사용자 · 감사 로그

User Requirement Audit
01

기능 개요

RMS 사용자가 시스템에서 수행한 모든 행위(로그인·로그아웃·설정 변경·계정 관리·권한 변경 등)의 시계열 이력을 조회하는 화면이다. 사용자 단위의 행위 추적과 감사 대응이 주된 목적이다.

Recipe History·Method History·Equipment History가 자료별 변경을 다룬다면, RMS Log는 사용자별 행위를 다룬다. "누가 시스템에 무엇을 했는가"를 추적하는 핵심 화면이다.

02

사용자 니즈

주요 사용자

  • RMS 관리자 — 시스템 사용 현황 점검
  • Vendor — 보안 감사 및 비정상 행위 추적
  • 감사 담당자 — 사용자 행위 검증

Pain Point (RMS 도입 전)

  • 사용자 행위가 시스템에 기록되지 않거나 분산되어 있음
  • 비정상 접근 시도를 사후에 추적하기 어려움
  • 감사 대응 시 자료 수집에 많은 시간 소요

기대 효과

  • 모든 사용자 행위의 자동 기록과 통합 조회
  • 비정상 행위 패턴의 사후 추적 가능
  • 감사 자료의 즉시 추출
03

기능 요구사항

■ FR-1. 사용 이력 조회

  • FR-1.1 화면 진입 시 사용자 행위 이력이 시계열(최신 우선)로 표시된다.
  • FR-1.2 각 항목에는 발생 시각, 사용자, 액션 유형, 대상, 결과(성공·실패), 상세 내용이 표시된다.
  • FR-1.3 액션 유형은 Login / Logout / Login Failed / Setting Change / Account Manage / Privilege Change 등을 포함한다.
  • FR-1.4 페이지네이션 또는 가상 스크롤로 대량 데이터를 효율적으로 표시한다.

■ FR-2. 사용자별 · 액션별 필터

  • FR-2.1 사용자는 특정 사용자의 행위만 추출할 수 있어야 한다.
  • FR-2.2 사용자는 특정 액션 유형만 추출할 수 있어야 한다.
  • FR-2.3 사용자는 결과(성공·실패)로 필터링할 수 있어야 한다.
  • FR-2.4 다중 필터를 동시 적용할 수 있다(AND 조건).

■ FR-3. 시간 범위 검색

  • FR-3.1 사용자는 시작 시각과 종료 시각을 지정하여 해당 범위의 이력만 조회할 수 있어야 한다.
  • FR-3.2 단축 옵션(오늘 / 어제 / 최근 7일 / 최근 30일)을 제공한다.

■ FR-4. Export

  • FR-4.1 사용자는 현재 필터링 결과를 CSV 또는 Excel로 Export할 수 있어야 한다.
  • FR-4.2 Export 파일은 SEMI 감사 대응 포맷에 맞춰 컬럼 구성과 메타가 포함된다.

■ FR-5. 데이터 보존

  • FR-5.1 사용 이력은 어떤 사용자도 수정·삭제할 수 없다(append-only).
  • FR-5.2 이력 보존 기간은 최소 3년이다.
  • FR-5.3 비활성화된 계정의 이력도 보존된다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 화면 진입 후 초기 데이터 표시까지 5초 이내.
  • NFR-1.2 필터 적용 응답은 3초 이내.

■ NFR-2. 권한

  • NFR-2.1 RMS Log 조회는 Vendor, Admin만 가능하다.
  • NFR-2.2 Export는 Vendor, Admin이 가능하다.
  • NFR-2.3 일반 사용자(Engineer, Operator)는 RMS Log에 접근할 수 없다.

■ NFR-3. 데이터 무결성

  • NFR-3.1 이력은 append-only로 관리된다.
  • NFR-3.2 이력 자체에 대한 조회 행위도 RMS Log에 기록된다(감사 대응).
05

제약사항 · 정책

  • 정책 RMS Log는 모든 사용자 행위(성공·실패 모두)를 기록한다. 로그인 시도 실패도 기록 대상이다.
  • 정책 이력은 append-only이며 어떤 경로에서도 수정·삭제할 수 없다.
  • 정책 이력 보존 기간은 최소 3년이다.
  • 정책 RMS Log 조회 행위 자체도 RMS Log에 기록된다. 감사 자료가 누구에게 노출되었는지 추적 가능해야 한다.
  • 정책 비활성화된 계정의 이력도 보존된다. 이력 추적 가능성이 우선이다.
  • 제약 RMS Log 조회·Export 권한은 Vendor, Admin로 한정된다.
06

관련 Test Cases

  • TC_HST_RMS_002 — 사용자별 · 액션별 필터 (FR-2)
  • TC_HST_RMS_003 — 시간 범위 검색 (FR-3)
  • TC_HST_RMS_004 — Log Export 기능 (FR-4)
