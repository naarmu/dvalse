TC_EQP_GRP_004

그룹에 장비 추가 · 제거

Functional High
01

Precondition

  1. 그룹 GROUP_A가 존재함 (현재 소속: EQ_001)
  2. 다른 그룹 GROUP_B에 EQ_002가 소속됨
  3. 그룹에 속하지 않은 EQ_003이 존재
02

Test Steps

  1. GROUP_A 선택
  2. Select Equipment(multiple) 영역에서 EQ_003 추가 선택
  3. Update 클릭 → GROUP_A의 멤버로 EQ_001, EQ_003이 되는지 확인
  4. GROUP_A 재선택 후 EQ_001의 선택 해제
  5. Update 클릭 → GROUP_A의 멤버에서 EQ_001이 빠지는지 확인
  6. EQ_002를 GROUP_A에 추가 → GROUP_B에서 EQ_002가 자동으로 빠지는지 확인
03

Test Data

Initial: GROUP_A: [EQ_001] GROUP_B: [EQ_002] 미지정: [EQ_003] Final (목표): GROUP_A: [EQ_002, EQ_003] GROUP_B: [] 미지정: [EQ_001]
04

Expected Result

  1. 장비 추가·제거가 즉시 반영됨
  2. 한 장비가 다른 그룹에 새로 배정되면 이전 그룹에서 자동으로 제거됨 (단일 그룹 소속 원칙)
  3. 그룹에서 제거된 장비는 "미지정" 상태로 복귀 (삭제되지 않음)
  4. Equipment Manager 트리 및 Dashboard에 즉시 반영
  5. 변경 이력이 Server Log에 기록됨
05

비고 (QA · 개발 메모)

단일 vs 다중 소속: 본 TC는 "한 장비는 한 그룹에만 소속 가능" 정책을 가정. 만약 다중 그룹 소속을 허용한다면 기대 결과를 수정해야 함.

RMS UI 확인: RMS 화면에서 Select Equipment(multiple) 영역과 "Selected - 6" 표시가 있음. 이 카운트가 실시간 갱신되는지 확인.

06

제안 사항

UX 개선 장비를 그룹에 추가할 때 드래그앤드롭으로 다른 그룹에서 이동시킬 수 있으면 편합니다. 특히 여러 장비를 재배치할 때 현재의 "각 그룹 진입 → 체크/해제" 방식보다 훨씬 빠릅니다.

기획 검토 단일 그룹 소속 원칙이 맞는지 재확인이 필요합니다. 장비가 "9F"와 "Clean Room"처럼 물리적 위치 + 기능 분류 두 축으로 관리될 가능성도 고려.
