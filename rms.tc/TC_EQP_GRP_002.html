TC_EQP_GRP_002

그룹명 중복 차단

Negative Medium
01

Precondition

  1. Vendor 또는 Admin 계정으로 로그인됨
  2. 기존에 GROUP_A 그룹이 존재함
02

Test Steps

  1. Edit Equipment Groups 화면에서 Add Group 클릭
  2. Group Name에 GROUP_A 입력 (중복)
  3. Save 클릭
  4. 에러 응답 확인
  5. 대소문자만 다른 경우도 테스트 (예: group_a)
03

Test Data

Case 1: "GROUP_A" (완전 중복) Case 2: "group_a" (대소문자 차이) Case 3: " GROUP_A " (공백 포함)
04

Expected Result

  1. 완전 중복 시 생성이 차단되고 에러 메시지 표시 (예: "이미 존재하는 그룹명입니다")
  2. 대소문자 차이에 대한 정책이 적용됨 (일반적으로 대소문자 무관으로 중복 판단 권장)
  3. 앞뒤 공백은 자동 제거(trim) 후 중복 검사 수행
  4. 입력 필드 값은 유지되어 사용자가 수정 후 재시도 가능
05

비고 (QA · 개발 메모)

대소문자 정책: 그룹명이 대소문자를 구분하는지 여부는 명시적 정의 필요. 운영 혼선 방지를 위해 대소문자 무관 중복 판단 권장.

06

제안 사항

UX 개선 Group Name 필드에 포커스 아웃 시 실시간 중복 검사를 수행하면 Save 누르기 전에 피드백을 받을 수 있습니다.

기획 검토 이미 삭제된 그룹과 동일한 이름을 재사용할 수 있는지 여부도 정책 확인이 필요합니다. (History 일관성 관점에서 중요)
