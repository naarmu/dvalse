TC_EQP_REG_005

등록 후 Agent 미설치 장비의 미연결 상태

Functional High
01

Precondition

  1. Equipment ID EQ_NEW_AGT가 신규 등록되었음
  2. 해당 장비 PC에는 RMS Agent가 설치되지 않은 상태
02

Test Steps

  1. Equipment Manager에서 EQ_NEW_AGT 확인
  2. 연결 상태 아이콘 확인 (Offline 또는 "Agent Not Installed" 상태)
  3. 우측 상세 영역에서 Agent Version 필드 확인 (빈 값 또는 "N/A")
  4. 해당 장비에 Recipe 배포를 시도 → 배포가 차단되는지 확인
  5. 10분 이상 시간 경과 후에도 상태가 Offline을 유지하는지 확인
03

Test Data

Equipment: EQ_NEW_AGT (등록됨, Agent 미설치)
04

Expected Result

  1. 장비 상태가 Offline 또는 "Agent Not Installed" 등 구분 가능한 상태로 표시
  2. Agent Version 등 Agent 관련 정보 필드는 빈 값 또는 "—" 로 표시
  3. 해당 장비를 대상으로 하는 Recipe 배포 시도 시 차단되고 적절한 에러 메시지 표시
  4. 시간이 경과해도 자동으로 Online으로 전환되지 않음 (Agent가 실제로 설치/실행되기 전까지)
  5. 장비 등록 이력은 유지됨
05

비고 (QA · 개발 메모)

상태 구분: "Offline(등록은 됐고 Agent는 있으나 꺼짐)" 과 "Agent 미설치(아예 Agent가 없음)"를 UI에서 구분 표시하는지 확인 필요. RMS 슬라이드에서는 Disconnected로 통일되어 보이지만, 원인 구분이 되면 운영 편의성이 크게 올라갑니다.

06

제안 사항

UX 개선 Agent 미설치 장비에 마우스오버 시 "이 장비에 Agent를 설치해야 연결됩니다" + Installer 다운로드 링크를 tooltip으로 제공하면 운영자 입장에서 다음 액션이 분명해집니다.

상태 구분 상태를 Online / Offline(연결 끊김) / Not Installed(Agent 미설치) 3단계로 나누는 것을 제안드립니다. 현재 Disconnected 하나로 통합되어 있는 상태에서 운영자가 원인을 파악하기 어렵습니다.
