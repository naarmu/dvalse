TC_MTD_DEP_008

배포 결과 장비별 성공 · 실패 구분 표시

Functional High
01

Precondition

  1. 4대 장비에 배포 — 일부는 성공, 일부는 실패 발생하도록 환경 조성 (1대 네트워크 차단)
02

Test Steps

  1. Method 배포 실행
  2. 완료 후 결과 화면에서 장비별 성공/실패 구분 표시 확인
  3. 실패 장비의 상세 실패 사유 조회 가능 여부 확인
  4. 실패 장비만 대상으로 "재시도" 기능 확인
03

Test Data

Targets: EQ_001, 002, 003 (Online) / EQ_004 (Offline 강제) Expected: 3 성공, 1 실패
04

Expected Result

  1. 결과 요약에 "성공 3 / 실패 1" 표시
  2. 장비별 결과가 명확히 구분됨 (성공=체크, 실패=X 또는 !)
  3. 실패 장비 클릭 시 상세 실패 사유 확인 가능
  4. 실패한 장비만 선택해서 재시도 가능
05

비고 (QA · 개발 메모)

Recipe Deploy와 동일 UI 패턴: TC_RCP_DEP_008과 같은 구조여야 사용자 학습 비용이 낮음.

재시도 시나리오: 실패 원인 해결 후(네트워크 복구) "재시도" 버튼 한 번으로 재배포 가능해야 함.

06

제안 사항

일관성 Recipe와 Method의 배포 결과 화면이 동일한 UI를 사용하도록 통일 권장합니다.

UX 개선 결과 화면에서 "실패한 것만 재시도" 원클릭 버튼 권장.
