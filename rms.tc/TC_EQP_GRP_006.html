TC_EQP_GRP_006

장비가 속한 그룹 삭제 시 경고 및 처리

Negative High
01

Precondition

  1. Vendor 또는 Admin 계정으로 로그인됨
  2. 장비가 2대 이상 속한 그룹 GROUP_A가 존재함
  3. 소속 장비: EQ_001, EQ_002
02

Test Steps

  1. Edit Equipment Groups 화면 진입
  2. 그룹 목록에서 GROUP_A 선택
  3. Delete Group 버튼 클릭
  4. 표시되는 확인 팝업 내용 확인
  5. 팝업에서 확인 선택
  6. 완료 후 장비 목록에서 EQ_001, EQ_002의 그룹 속성 확인
03

Test Data

Group: GROUP_A Member Equipments: EQ_001, EQ_002 Login: admin_test (Admin Role)
04

Expected Result

  1. 삭제 버튼 클릭 시 경고 팝업이 표시되고, 소속된 장비가 있음이 명시됨
    (예: "해당 그룹에는 2대의 장비가 속해있습니다. 삭제 시 장비는 그룹 미지정 상태가 됩니다. 계속하시겠습니까?")
  2. 확인 선택 시:
    • 그룹 GROUP_A가 삭제됨
    • EQ_001, EQ_002는 삭제되지 않고 "그룹 미지정" 상태로 복귀됨
  3. 취소 선택 시: 그룹과 장비 모두 변경 없이 유지됨
  4. 그룹 삭제 이력이 Server Log에 기록됨 (삭제 주체 계정, 시각, 소속 장비 수 포함)
05

비고 (QA · 개발 메모)

정책 재확인 필요: 이 TC는 "장비 자체는 보존, 그룹 소속만 해제"를 가정하고 작성되었음. 실제 구현이 "삭제 차단(소속 장비가 있으면 삭제 불가)" 정책이라면 기대 결과를 수정해야 함.

DevExpress 관련: 확인 팝업은 DXMessageBox 또는 자체 Dialog일 수 있음. 팝업 닫기(X 버튼)와 배경 클릭 시 동작이 "취소"와 동일해야 함.

06

제안 사항

정책 확정 "소속 장비가 있는 그룹 삭제"에 대한 정책을 명시적으로 확정해주셔야 합니다. 현재 제가 권장드리는 방향은:

  • 기본: 경고 후 허용 — 장비를 "미지정" 그룹으로 이동시키고 그룹만 삭제
  • 대안: 삭제 차단 — 먼저 장비를 다른 그룹으로 옮기거나 그룹에서 빼라고 안내

반도체 공정 환경에서는 전자(경고 후 허용)가 더 실용적입니다. "미지정" 장비가 일시적으로 생기더라도 장비 자체는 계속 동작하는 것이 중요하고, 장비를 다른 그룹으로 일일이 옮기는 것은 운영상 번거롭습니다.

UX 개선 경고 팝업 안에 소속 장비 목록을 축약 표시(예: "EQ_001 외 1대")하여 사용자가 실수로 삭제하는 것을 방지하는 안내가 있으면 좋겠습니다.
