TC_RCP_DIR_002

잘못된 확장자 · 손상된 파일 업로드 거부

Negative Critical
01

Precondition

  1. 테스트용 잘못된 파일 준비
02

Test Steps

  1. .psr이 아닌 파일 업로드 시도 (예: test.txt)
  2. .psr이지만 깨진 바이너리 파일 업로드
  3. 각 케이스별 에러 확인
03

Test Data

Case 1: test.txt (확장자 불일치) Case 2: corrupt.psr (바이너리 손상)
04

Expected Result

  1. Case 1: 확장자 체크로 즉시 차단
  2. Case 2: 서버 측 파일 검증으로 차단, "파일이 손상되었거나 올바른 .psr 형식이 아닙니다"
  3. 서버에 파일이 저장되지 않음
  4. 사용자에게 명확한 에러 메시지
05

비고 (QA · 개발 메모)

파일 검증: 확장자 + 매직 바이트 + 스키마 검증 3단계 방어 필요.

06

제안 사항

보안 Direct Upload는 악의적 파일 유입 경로가 될 수 있으므로 파일 스캔(안티바이러스 연동)도 고려해주세요. 특히 장비로 재배포될 경우 위험이 확산됩니다.
