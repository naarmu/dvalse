URS_HST_RCP

Recipe 이력

User Requirement Audit
01

기능 개요

Recipe와 관련된 모든 이벤트(등록·수집·배포·삭제·Master 변경 등)의 시계열 이력을 조회하는 화면이다. 사용자는 특정 Recipe가 언제 어떻게 변경되었는지 추적하여, 공정 변경의 사후 검증이나 감사 자료로 활용한다.

Recipe Manager가 현재 상태를 보여준다면, Recipe History는 그 상태에 도달한 과정을 보여준다. Master 변경 같은 중요한 결정의 이력을 보존하는 핵심 화면이다.

02

사용자 니즈

주요 사용자

  • 공정 표준화 팀 — Recipe 변경 사유와 시점 추적
  • 공정 엔지니어 — 특정 Recipe의 변경 이력 검토
  • 감사 담당자 — Recipe 운영 이력 검증

Pain Point (RMS 도입 전)

  • Recipe 변경이 언제, 누가, 왜 이루어졌는지 기록되지 않음
  • 공정 사고 발생 시 직전 Recipe 변경 이력 추적 곤란

기대 효과

  • Recipe 모든 변경의 자동 기록과 시계열 조회
  • 이벤트 타입별 필터로 원하는 변경만 추출
  • 특정 Recipe 단위로 변경 이력 추적 가능
03

기능 요구사항

■ FR-1. 이력 목록 조회

  • FR-1.1 화면 진입 시 Recipe 관련 이벤트 이력이 시계열(최신 우선)로 표시된다.
  • FR-1.2 각 항목에는 발생 시각, Recipe명, 버전, 이벤트 유형, 사용자, 상세 내용이 표시된다.
  • FR-1.3 이벤트 유형은 Register / Collected / Deployed / Master Change / Updated / Deleted를 포함한다.
  • FR-1.4 페이지네이션 또는 가상 스크롤로 대량 데이터를 효율적으로 표시한다.

■ FR-2. 필터링

  • FR-2.1 사용자는 날짜 범위, 이벤트 타입, 사용자, Recipe명으로 필터링할 수 있어야 한다.
  • FR-2.2 다중 필터를 동시 적용할 수 있다(AND 조건).

■ FR-3. Recipe별 이력 추적

  • FR-3.1 사용자는 특정 Recipe를 선택하여 그 Recipe의 모든 이력만 추출할 수 있어야 한다.
  • FR-3.2 Recipe Manager의 상세 화면에서 "이력 보기" 진입점을 제공하여, 해당 Recipe로 자동 필터링된 History 화면으로 이동한다.
  • FR-3.3 이력 항목 중 변경된 버전은 클릭 시 해당 버전 비교 화면으로 이동할 수 있다.

■ FR-4. Export

  • FR-4.1 사용자는 현재 필터링 결과를 CSV 또는 Excel로 Export할 수 있어야 한다.
  • FR-4.2 Export 파일은 SEMI 감사 대응 포맷에 맞춰 컬럼 구성과 메타가 포함된다.

■ FR-5. 데이터 보존

  • FR-5.1 이력 데이터는 어떤 사용자도 수정·삭제할 수 없다(append-only).
  • FR-5.2 이력 보존 기간은 최소 3년이다.
  • FR-5.3 삭제된 Recipe의 이력도 보존되며 조회 가능해야 한다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 화면 진입 후 초기 데이터 표시까지 5초 이내.
  • NFR-1.2 필터 적용 응답은 3초 이내.

■ NFR-2. 권한

  • NFR-2.1 이력 조회는 모든 사용자(All Roles)가 가능하다.
  • NFR-2.2 Export는 Vendor, Admin, Engineer가 가능하다.

■ NFR-3. 데이터 무결성

  • NFR-3.1 이력은 append-only로 관리된다.
  • NFR-3.2 Recipe가 삭제되어도 이력은 보존된다.
05

제약사항 · 정책

  • 정책 모든 이력은 append-only로 관리된다.
  • 정책 이력 보존 기간은 최소 3년이다.
  • 정책 Recipe가 삭제되어도 그 이력은 보존된다. 이력 추적 가능성을 위해서이다.
  • 정책 Recipe Manager에서 "이력 보기" 진입점을 제공한다. 작업 흐름의 자연스러운 연결을 위함이다.
  • 제약 Export는 Vendor, Admin, Engineer로 한정된다.
  • 제약 Export 파일은 SEMI 감사 대응 포맷에 맞춘 컬럼 구성을 가진다.
06

관련 Test Cases

  • TC_HST_RCP_001 — Recipe 이력 조회 (등록 · 수집 · 배포 · 변경) (FR-1)
  • TC_HST_RCP_002 — 이벤트 타입별 필터링 (FR-2)
  • TC_HST_RCP_003 — Recipe별 이력 추적 (FR-3)
  • TC_HST_RCP_004 — 이력 Export 기능 (FR-4)
