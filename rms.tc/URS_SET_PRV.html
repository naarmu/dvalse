URS_SET_PRV

권한 관리

User Requirement Security
01

기능 개요

RMS의 4단계 Role(Vendor / Admin / Engineer / Operator)에 대해 어떤 화면과 액션을 사용할 수 있는지 설정하는 화면이다. 사용자별 개별 권한이 아니라 Role 단위의 권한 정책을 관리한다.

Privilege는 RMS의 보안 골격을 정의하는 화면이다. 한 번 설정된 Role 권한은 시스템 전체에 일관되게 적용되며, 모든 사용자의 행위가 자신의 Role 정책에 따라 제약된다.

02

사용자 니즈

주요 사용자

  • Vendor — RMS 도입 시 Role별 권한 정책 정의
  • RMS 관리자 — 운영 중 권한 정책 조정

Pain Point (RMS 도입 전)

  • 장비별 로컬 권한 관리로 정책이 일관되지 않음
  • 권한 변경 시 모든 장비를 일일이 갱신해야 함
  • "누가 무엇을 할 수 있는지"가 시스템에 명시되지 않음

기대 효과

  • Role 기반의 일관된 권한 정책 적용
  • 중앙 설정으로 모든 사용자에게 동시 반영
  • 권한 변경 이력의 자동 기록
03

기능 요구사항

■ FR-1. Role 목록 표시

  • FR-1.1 Privilege 화면에는 시스템에 정의된 4개 Role(Vendor / Admin / Engineer / Operator)이 표시된다.
  • FR-1.2 각 Role별로 화면 접근 권한과 액션 권한 설정이 표시된다.
  • FR-1.3 각 권한 항목에는 현재 설정 값(허용 / 차단)과 기본값이 함께 표시된다.

■ FR-2. 화면 접근 권한 설정

  • FR-2.1 Vendor는 각 Role별로 어떤 화면을 표시할지 설정할 수 있어야 한다.
  • FR-2.2 비활성화된 화면은 해당 Role 사용자의 메뉴에서 숨김 또는 비활성으로 표시된다.
  • FR-2.3 핵심 보안 관련 화면(예: Privilege, Account)은 기본적으로 Vendor만 접근 가능하다.

■ FR-3. 액션 권한 설정

  • FR-3.1 Vendor는 각 Role별로 파괴적 액션(삭제·배포·수정 등)의 수행 권한을 설정할 수 있다.
  • FR-3.2 권한이 없는 사용자에게는 해당 액션 버튼이 비활성 또는 숨김 처리된다.
  • FR-3.3 UI 제약과 함께 서버 API 단계의 권한 검증이 이중으로 수행된다.

■ FR-4. 설정 반영

  • FR-4.1 권한 설정 변경은 저장 시 즉시 서버에 반영된다.
  • FR-4.2 변경된 설정은 새로 로그인하는 사용자에게 즉시 적용된다.
  • FR-4.3 이미 로그인 중인 사용자에게는 다음 화면 전환 시점부터 반영된다.
  • FR-4.4 권한 변경으로 사용자가 현재 보고 있는 화면에 접근할 수 없게 된 경우, 즉시 다른 화면으로 안내한다.

■ FR-5. Vendor 자기 권한 보호

  • FR-5.1 Vendor는 자신의 Role 권한을 축소할 수 없다.
  • FR-5.2 마지막 남은 Vendor 계정의 권한 변경은 차단된다(시스템 잠금 방지).

■ FR-6. 변경 이력

  • FR-6.1 권한 설정의 모든 변경(허용 → 차단, 차단 → 허용)은 RMS Log에 자동 기록된다.
  • FR-6.2 기록 항목에는 변경자, 변경 시각, 대상 Role, 변경 내용이 포함된다.
  • FR-6.3 변경 이력은 수정·삭제할 수 없다.
04

비기능 요구사항

■ NFR-1. 권한

  • NFR-1.1 Privilege 화면 자체는 Vendor만 접근 가능하다.
  • NFR-1.2 다른 Role은 Privilege 화면 메뉴 자체가 표시되지 않아야 한다.

■ NFR-2. 보안

  • NFR-2.1 권한 검증은 UI 제약(메뉴 숨김, 버튼 비활성)과 서버 API 검증의 이중 방어로 수행한다.
  • NFR-2.2 사용자가 직접 URL 또는 API를 호출해도 권한 없는 작업은 서버에서 거부된다.

■ NFR-3. 가용성

  • NFR-3.1 마지막 Vendor 계정의 권한이 축소되어 시스템에 접근 불가능한 상태가 되지 않도록 한다.
05

제약사항 · 정책

  • 정책 RMS의 Role은 Vendor / Admin / Engineer / Operator 4단계로 고정된다. 운영 중 추가·삭제는 지원하지 않는다.
  • 정책 Privilege 화면은 Vendor만 접근 가능하다. 보안 정책의 최상위 결정 권한이 명확히 분리되도록 한다.
  • 정책 권한 변경은 새로 로그인하는 사용자에게 즉시 적용되며, 기존 로그인 세션에는 다음 화면 전환 시점부터 반영된다.
  • 정책 Vendor는 자기 자신의 권한을 축소할 수 없다. 시스템 잠금 방지가 목적이다.
  • 정책 마지막 남은 Vendor 계정의 권한 변경은 차단된다.
  • 정책 권한 변경 이력은 자동 기록되며 수정·삭제할 수 없다.
  • 제약 권한 검증은 UI와 서버 API 양쪽에서 이중으로 수행한다.
  • 제약 권한 설정은 Role 단위로만 관리한다. 사용자별 개별 권한 부여는 지원하지 않는다.
