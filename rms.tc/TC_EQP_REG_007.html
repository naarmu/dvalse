TC_EQP_REG_007

장비 등록 취소 동작

Functional Medium
01

Precondition

  1. Register Equipment 화면 진입
  2. 아직 서버에 Save되지 않은 상태
02

Test Steps

  1. Equipment ID, Name 등 몇 개 필드에 값을 입력한 상태
  2. Cancel 또는 취소 버튼 클릭 (또는 화면 이탈 시도)
  3. 확인 팝업 유무 확인
  4. 팝업에서 "예" 선택 시 폼 초기화 또는 화면 이탈 확인
  5. Equipment Manager 목록에 해당 장비가 등록되지 않았음을 확인
03

Test Data

입력 중인 데이터: Equipment ID: EQ_TEMP Name: Temp Equipment
04

Expected Result

  1. 입력 도중 취소 시도 시 "작성 중인 내용이 사라집니다" 등의 확인 팝업 표시
  2. "예" 선택 시: 폼이 초기화되고 장비는 등록되지 않음
  3. "아니오" 선택 시: 현재 폼 상태가 유지됨
  4. 확인 팝업은 실제로 입력값이 있을 때만 표시 (빈 폼 상태에서는 팝업 없이 바로 닫기)
  5. Equipment Manager 목록에 해당 장비가 추가되지 않았음
05

비고 (QA · 개발 메모)

Unsaved 감지: 브라우저/앱 종료 시(창 닫기)에도 확인 팝업을 띄울지 정책 확인 필요. WPF에서는 Window Closing 이벤트로 처리 가능.

06

제안 사항

UX 개선 사용자가 실수로 화면을 이탈해도 입력 중이던 값을 임시 저장(Draft)해두고, 다시 들어왔을 때 복원을 제안하는 기능이 있으면 편리합니다. 다만 장비 등록은 자주 하는 작업이 아니므로 우선순위는 낮습니다.
