TC_MTD_MGR_008

Method 삭제 기능 (권한 · 확인 팝업)

Functional High
01

Precondition

  1. 삭제 대상 Method "TEST_METHOD"가 서버에 존재 (버전 2개 포함)
  2. 해당 Method가 어떤 장비에도 현재 배포되어 있지 않음
02

Test Steps

  1. Method Manager에서 "TEST_METHOD" 선택
  2. 우측 액션 영역의 Delete 버튼 클릭
  3. 확인 팝업 내용 확인 (버전 수 포함 여부 등)
  4. "확인" 선택
  5. Method가 목록에서 제거되고 History에 삭제 이력 기록 확인
  6. 동일 시나리오로 Operator 계정에서 시도 → Delete 버튼 비활성/숨김 확인
03

Test Data

Method: TEST_METHOD (v1, v2 포함) Login 1: admin_test (Admin) — 삭제 성공 Login 2: operator_test (Operator) — 삭제 차단
04

Expected Result

  1. 확인 팝업에 Method 이름, 버전 수, 영향 안내가 명시됨
  2. "확인" 시 Method와 모든 버전이 서버에서 삭제
  3. Method History에 삭제 이력이 기록 (삭제자, 시각, 버전 수)
  4. "취소" 시 변경 없음
  5. Operator 계정에서는 Delete 버튼 자체가 비활성 또는 숨김
  6. 현재 어떤 장비에 배포된 Method는 삭제 시도 시 경고 또는 차단 (정책 확인 필요)
05

비고 (QA · 개발 메모)

Recipe와 동일 정책: Recipe 삭제(TC_RCP_MGR_009)와 같은 정책을 적용해야 일관성 유지.

정책 확정 필요: 현재 배포 중인 Method의 삭제 허용 여부.

06

제안 사항

일관성 Recipe와 Method의 삭제 정책을 통일해야 합니다. 권장: 현재 배포 중인 Method는 경고 후 허용 + 배포된 장비 목록 명시.

데이터 무결성 Recipe와 동일하게 소프트 삭제(Soft Delete)를 권장드립니다. 30일 복원 가능하도록.

UX 개선 파괴적 액션이므로 확인 팝업의 "삭제" 버튼은 빨간색, 기본 포커스는 "취소"에 두는 것을 제안드립니다.
