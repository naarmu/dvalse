URS_EQP_GRP

장비 그룹 관리

User Requirement Organization
01

기능 개요

여러 장비를 의미 있는 단위로 묶어 그룹으로 관리하는 화면이다. 사용자는 장비를 위치(층·라인), 용도, 모델 등 다양한 기준으로 분류하여 그룹을 만들고, Recipe·Method 배포 시 그룹 단위로 일괄 작업할 수 있다.

장비 그룹은 Recipe 그룹·Method 그룹과는 다른 의미를 갖는다. 한 장비는 단일 그룹에만 소속되며, 그룹 트리는 운영 위계(예: FAB → 라인 → 장비) 표현에 가깝다.

02

사용자 니즈

주요 사용자

  • RMS 관리자 — FAB·라인별 장비 자산 분류
  • 장비 엔지니어 — 그룹 단위 일괄 배포 작업
  • 공정 표준화 팀 — 동일 공정 라인의 장비 묶음 관리

Pain Point (RMS 도입 전)

  • 장비가 많아질수록 개별 선택만으로는 작업 효율이 떨어짐
  • 장비 그룹 정보가 시스템에 명시되지 않아 운영자 머릿속에만 존재
  • 일괄 작업 시 누락·오선택 사고가 발생

기대 효과

  • 장비를 운영 위계에 맞춰 분류 가능
  • 그룹 단위 일괄 배포로 작업 효율 향상
  • 그룹 트리를 통한 시각적 자산 조감
03

기능 요구사항

■ FR-1. 그룹 생성

  • FR-1.1 사용자는 새 장비 그룹을 생성할 수 있어야 한다.
  • FR-1.2 그룹 생성 시 Group Name과 초기 멤버 장비를 선택할 수 있다.
  • FR-1.3 동일한 이름의 그룹은 생성할 수 없다. 중복 시 명확한 에러 메시지가 표시된다.
  • FR-1.4 그룹명 중복 검사는 대소문자를 구분하지 않는다.

■ FR-2. 멤버 편집

  • FR-2.1 사용자는 기존 그룹에 장비를 추가하거나 제거할 수 있어야 한다.
  • FR-2.2 드래그 앤 드롭 방식으로 장비 이동을 지원한다.
  • FR-2.3 장비를 새 그룹으로 이동하면 이전 그룹에서 자동 제거된다.
  • FR-2.4 멤버 변경 사항은 다른 화면(Equipment Manager 등)에 즉시 반영된다.

■ FR-3. 그룹 정보 변경

  • FR-3.1 사용자는 그룹 이름을 변경할 수 있어야 한다.
  • FR-3.2 이름 변경 시 변경 이력이 기록된다.
  • FR-3.3 이름 변경은 다른 화면에 즉시 반영된다.

■ FR-4. 그룹 삭제

  • FR-4.1 사용자는 빈 그룹(소속 장비 0대)을 삭제할 수 있어야 한다.
  • FR-4.2 소속 장비가 있는 그룹을 삭제하려는 경우, 사용자에게 경고가 표시되며 처리 방식을 선택하게 한다: 그룹만 삭제(장비는 미지정 상태로 복귀) / 취소.
  • FR-4.3 그룹 삭제 시 그룹만 제거되며, 소속 장비 자체는 유지된다.
  • FR-4.4 그룹 삭제 이력은 자동 기록된다.

■ FR-5. 다른 화면과의 연동

  • FR-5.1 그룹 정보는 Equipment Manager의 좌측 그룹 트리에 계층 구조로 표시된다.
  • FR-5.2 Deploy Recipes·Deploy Methods 화면에서 그룹 단위로 배포 대상을 선택할 수 있다.
  • FR-5.3 그룹 트리의 계층 표시는 모든 연동 화면에서 일관되어야 한다.
04

비기능 요구사항

■ NFR-1. 권한

  • NFR-1.1 그룹 생성·멤버 편집·이름 변경·삭제는 Vendor, Admin만 수행 가능하다.
  • NFR-1.2 일반 사용자는 그룹 정보를 조회할 수 있으나 변경할 수 없다.

■ NFR-2. 데이터 무결성

  • NFR-2.1 그룹 변경 이력(생성·수정·삭제)은 자동 기록되며 수정·삭제할 수 없다.
  • NFR-2.2 그룹 삭제 시 소속 장비는 절대 함께 삭제되지 않는다.

■ NFR-3. 성능

  • NFR-3.1 그룹 100개 이상 환경에서도 그룹 트리 표시가 3초 이내 완료되어야 한다.

■ NFR-4. UI 일관성

  • NFR-4.1 Equipment 그룹 관리 화면은 Recipe 그룹·Method 그룹 관리와 동일한 UX 패턴을 따른다.
  • NFR-4.2 Equipment 그룹과 Recipe·Method 그룹은 시각적으로 구분 표시된다.
05

제약사항 · 정책

  • 정책 한 장비는 단일 그룹에만 소속된다. (Recipe·Method 그룹의 다중 소속 정책과 의도적으로 다르다.) 장비 그룹은 운영 위계 표현에 가깝기 때문에 단일 소속이 적합하다.
  • 정책 그룹 삭제 시 소속 장비는 보존된다. 장비는 "미지정" 상태로 복귀하며, 다른 그룹에 재배치하기 전까지 그룹 트리에서는 별도로 표시된다.
  • 정책 그룹명 중복 검사는 대소문자를 구분하지 않는다.
  • 정책 그룹 변경 이력은 자동 기록되며 수정·삭제할 수 없다.
  • 제약 그룹 관리 권한은 Vendor, Admin로 한정된다.
  • 제약 Equipment 그룹은 Recipe 그룹·Method 그룹과 별개의 데이터 모델이며 상호 변환되지 않는다.
