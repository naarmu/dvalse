TC_RCP_CMP_004

3개 이상 Recipe 동시 비교

Functional High
01

Precondition

  1. 비교할 Recipe 3개 이상 존재
  2. 각 Recipe가 서로 일부 파라미터에서 차이를 가짐
02

Test Steps

  1. Compare Recipes에서 Recipe A, B, C를 각각 선택
  2. 비교 결과 테이블에 3개 컬럼으로 표시 확인
  3. 파라미터별 값을 세 Recipe에 걸쳐 비교
  4. Recipe 4개 이상 추가 시도 (허용 최대 개수 확인)
03

Test Data

Recipe A, B, C 각각 일부 파라미터 값 차이 보유
04

Expected Result

  1. 3개 이상 Recipe 선택 가능
  2. 비교 테이블이 선택한 개수만큼 컬럼 확장
  3. 한 행 내 3개 값이 모두 같으면 일반, 하나라도 다르면 하이라이트
  4. UI가 가로로 너무 길어지지 않도록 반응형 처리 (가로 스크롤 또는 축소)
  5. 최대 비교 개수가 제한되어 있다면 안내 메시지 표시
05

비고 (QA · 개발 메모)

최대 개수 정책: 실무에서 3~5개 비교가 주로 필요할 것. 10개 이상은 가독성이 급격히 떨어지므로 상한 필요.

06

제안 사항

기획 검토 최대 비교 가능 개수를 명시적으로 정의해주세요. 권장: 5개 (화면 가로폭 기준).

UX 개선 3개 이상 비교 시 기준 Recipe 지정 기능을 넣으면 유용합니다. 기준과 다른 값만 하이라이트되어 변화 추적이 쉬워집니다.
