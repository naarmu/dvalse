TC_RCP_MGR_002

Recipe 선택 시 상세 정보(버전·파라미터) 표시

Functional High
01

Precondition

  1. Recipe Manager 화면 진입
  2. 다수 버전을 가진 Recipe(예: Recipe "213" — v1, v2)가 존재
02

Test Steps

  1. 좌측 트리에서 Recipe "213" 선택
  2. 하위 버전 "v1 (2026-04-10 13:45)" 선택
  3. 중앙 Recipe Information 영역에 정보 표시 확인
  4. 우측 Recipe Contents 영역에 파라미터 값 표시 확인
  5. 다른 버전(v2)으로 선택 전환 → 정보가 해당 버전으로 갱신되는지 확인
03

Test Data

Recipe: 213 - v1 (2026-04-10 13:45) — Description: "Uploaded via RecipeUploadQueueService" - v2 (2026-04-13 11:17) Contents: Type=Wafer, Size=300, Die Based=TRUE, Die X/Y=20.0, Tip ID=NCHR ...
04

Expected Result

  1. 중앙 Recipe Information에 Recipe Type, Version, Last Saved, Collected Equipment, Description 표시
  2. 우측 Recipe Contents에 파라미터 목록(Type, Size, Thickness, Die Based, Die X/Y, Tip ID 등) 표시
  3. 버전 전환 시 두 영역 모두 새 버전 기준으로 갱신
  4. Description의 수정 버튼(연필 아이콘)이 권한에 맞게 표시
  5. Deployment History 영역도 해당 Recipe/버전 기준으로 표시
05

비고 (QA · 개발 메모)

RMS UI 참고: slide-27(Recipe Manager)에서 본 구성과 일치해야 함.

파라미터 개수: 표시되는 파라미터가 Manage Recipe Parameters에서 설정한 표시/숨김 상태에 따라 달라지는지 확인.

06

제안 사항

UX 개선 버전 전환 시 변경된 파라미터에 하이라이트를 주는 것을 고려해볼 수 있습니다. Compare Recipes 화면까지 가지 않고도 간단한 차이를 인지할 수 있어 편리합니다.

기획 검토 Description 필드는 자유 텍스트인데, Recipe 변경 사유를 구조화된 형태(예: Change Reason 카테고리 선택)로 받으면 이력 추적 시 유용합니다.
