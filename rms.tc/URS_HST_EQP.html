URS_HST_EQP

장비 이력

User Requirement Audit
01

기능 개요

장비에서 발생한 운영 이벤트(연결·연결 끊김·알람·등록·삭제 등)의 시계열 이력을 조회하는 화면이다. 사용자는 특정 장비가 언제 어떤 상태 변화를 겪었는지 추적하여, 운영 문제의 원인을 분석하거나 감사 자료로 활용한다.

Equipment History는 장비 단위의 이력을 다룬다. Recipe·Method 변경 이력은 Recipe History·Method History에서 별도 관리하며, 시스템 전반의 사용자 행위는 RMS Log에서 다룬다.

02

사용자 니즈

주요 사용자

  • 장비 엔지니어 — 장비 이상 발생 시 원인 추적
  • RMS 관리자 — 장비 운영 현황의 사후 검토
  • 감사 담당자 — 장비 운영 이력 검증

Pain Point (RMS 도입 전)

  • 장비별 이벤트 로그가 분산되어 통합 추적 어려움
  • 이상 발생 시점 전후의 상태 변화를 시계열로 확인할 수단이 없음
  • 감사 대응 시 자료 수집에 많은 시간 소요

기대 효과

  • 모든 장비 이벤트를 단일 화면에서 시계열로 조회
  • 다양한 필터로 원하는 시점·장비·이벤트만 추출
  • Export 기능으로 외부 자료로 활용 가능
03

기능 요구사항

■ FR-1. 이력 목록 조회

  • FR-1.1 화면 진입 시 장비 이벤트 이력이 시계열(최신 우선)로 목록 형태로 표시된다.
  • FR-1.2 각 항목에는 발생 시각, 장비명, 이벤트 유형(Connect / Disconnect / Alarm / Register / Update / Delete), 상세 내용이 표시된다.
  • FR-1.3 알람 이벤트는 별도 색상·아이콘으로 강조 표시한다.
  • FR-1.4 페이지네이션 또는 가상 스크롤로 대량 데이터를 효율적으로 표시한다.

■ FR-2. 필터링

  • FR-2.1 사용자는 날짜 범위, 특정 장비, 이벤트 유형으로 필터링할 수 있어야 한다.
  • FR-2.2 다중 필터를 동시 적용할 수 있다(AND 조건).
  • FR-2.3 필터 적용 결과의 건수가 화면에 표시된다.

■ FR-3. 검색

  • FR-3.1 사용자는 장비 ID, 이벤트 키워드 등으로 검색할 수 있어야 한다.
  • FR-3.2 검색은 부분 일치를 지원한다.

■ FR-4. Export

  • FR-4.1 사용자는 현재 필터링 결과를 CSV 또는 Excel로 Export할 수 있어야 한다.
  • FR-4.2 Export 파일에는 화면에 표시된 모든 컬럼이 포함된다.
  • FR-4.3 Export 파일명에는 생성 시각이 자동 포함된다.

■ FR-5. 데이터 보존

  • FR-5.1 이력 데이터는 어떤 사용자도 수정·삭제할 수 없다(append-only).
  • FR-5.2 이력 보존 기간은 최소 3년이다.
04

비기능 요구사항

■ NFR-1. 성능

  • NFR-1.1 화면 진입 후 초기 데이터 표시까지 5초 이내.
  • NFR-1.2 필터 적용 응답은 3초 이내.
  • NFR-1.3 Export는 10만 건 이하 데이터에서 30초 이내 완료.

■ NFR-2. 권한

  • NFR-2.1 이력 조회는 모든 사용자(All Roles)가 가능하다.
  • NFR-2.2 Export는 Vendor, Admin, Engineer가 가능하다.

■ NFR-3. 데이터 무결성

  • NFR-3.1 이력은 append-only로 관리되며 어떤 경로(UI, API)에서도 수정·삭제할 수 없다.
05

제약사항 · 정책

  • 정책 모든 이력은 append-only로 관리된다. 어떤 사용자도 수정·삭제할 수 없다.
  • 정책 이력 보존 기간은 최소 3년이다. (반도체 업계 감사 요구사항 기준)
  • 정책 알람 이벤트는 일반 이벤트와 시각적으로 구분 표시한다.
  • 정책 Export 결과의 정합성을 위해 필터 적용 상태와 시점이 파일에 함께 기록된다.
  • 제약 이력 조회는 모든 사용자에게 허용된다. Export는 Vendor, Admin, Engineer로 한정된다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 4건:

  • TC_HST_EQP_001 — 장비 이력 조회 및 목록 표시 (FR-1)
  • TC_HST_EQP_002 — 날짜 · 장비 · 상태별 필터링 (FR-2)
  • TC_HST_EQP_003 — 알람 이력 구분 표시 (FR-1.3)
  • TC_HST_EQP_004 — 이력 CSV · Excel Export (FR-4)
