TC_RCP_MGR_001

Recipe Manager 진입 및 Recipe 목록 로딩

Functional Critical
01

Precondition

  1. RMS Client가 실행되고 로그인된 상태
  2. 서버에 최소 5개 이상의 Recipe가 수집/등록되어 있음
  3. 각 Recipe는 1개 이상의 버전을 가짐
02

Test Steps

  1. 상단 메뉴에서 Recipe 클릭
  2. 하위 메뉴 Recipe Manager 클릭
  3. 화면 로딩 완료 대기 (최대 5초)
  4. 좌측 Recipe 트리, 중앙 Recipe Information, 우측 Recipe Contents 영역 확인
  5. 우측 액션 버튼(Deploy, Compare, Download, Upload, Delete) 표시 확인
03

Test Data

Recipes: 213, RMSRecipeWspinach_gnocci, SDC_A1WPP_GZ3_LENS_ADI 등 5개+
04

Expected Result

  1. 화면이 5초 이내 표시됨
  2. 좌측 트리에 등록된 모든 Recipe가 누락 없이 표시
  3. 각 Recipe 아래에 버전 목록(v1, v2...)이 펼침 가능한 형태로 표시
  4. 화면 진입 직후 선택된 Recipe가 없으면 중앙/우측 영역은 빈 상태(Empty State)로 표시
  5. 우측 액션 버튼은 Recipe 선택 여부에 따라 활성/비활성 상태 구분됨
05

비고 (QA · 개발 메모)

DevExpress: 좌측은 TreeListControl, 중앙·우측은 GridControl 기반으로 추정. 초기 로딩 시 트리 기본 펼침 depth 확인 필요.

성능: Recipe가 수백 개 쌓일 경우 초기 로딩 지연이 있을 수 있어 가상화 적용 여부 확인.

06

제안 사항

UX 개선 화면 진입 시 마지막으로 열었던 Recipe를 자동 선택하는 기능을 제안드립니다. Recipe Manager는 반복적으로 특정 Recipe를 확인하는 용도이므로 재선택 비용을 줄일 수 있습니다.

정보 구조 Recipe 개수가 많아질 경우 "즐겨찾기(Favorite)" 기능이 있으면 좋습니다. 자주 배포하는 Recipe를 상단에 고정할 수 있도록.
