TC_MTD_PRM_001

Method 파라미터 전체 목록 표시

Functional High
01

Precondition

  1. Method에 사용되는 파라미터가 여러 개 수집되어 서버 DB에 등록됨
02

Test Steps

  1. Manage Method Parameters 진입
  2. 수집된 모든 파라미터가 목록에 표시되는지 확인
  3. 각 파라미터의 속성(이름, 타입, 단위, 표시 여부) 확인
03

Test Data

Method 파라미터 예시: Scan Area, Scan Rate, Setpoint, Tip Mode, Feedback Gain, Filter, ...
04

Expected Result

  1. 파라미터 전체 목록 표시
  2. 각 항목에 이름, 데이터 타입, 단위, 현재 표시 상태(Show/Hide) 포함
  3. 정렬/검색 가능
05

비고 (QA · 개발 메모)

Recipe Parameter와 별개 관리: Method와 Recipe는 파라미터 체계가 서로 달라 별도 관리가 필요.

파라미터 갱신: 새로운 Method가 수집될 때 기존에 없던 파라미터가 발견되면 목록에 자동 추가되는지 확인.

06

제안 사항

정보 구조 Method 파라미터를 카테고리별로 그룹화(예: Scan, Feedback, Filter, Tip)하면 관리가 쉬워집니다.

기획 검토 Recipe와 Method의 파라미터 설정을 통합 관리할지, 분리할지 명확히 정의해야 합니다. 분리가 운영 자유도 측면에서 유리.
