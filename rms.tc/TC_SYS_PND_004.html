TC_SYS_PND_004

Pending 항목 무시 · 나중에 닫기 동작

Functional Medium
01

Precondition

  1. Pending 팝업 표시
02

Test Steps

  1. "나중에" 또는 "닫기" 버튼 클릭
  2. 다음 로그인 시 Pending 팝업이 다시 뜨는지 확인
  3. "이 항목 무시" 옵션이 있다면 해당 항목 제거 동작 확인
03

Test Data

닫기 액션
04

Expected Result

  1. "닫기" 시 팝업만 닫히고 Pending 항목은 유지
  2. "무시" 시 해당 항목이 Pending에서 제거 (서버 DB에도 반영)
  3. 다음 로그인 시 남은 Pending에 대해 팝업 재표시
05

비고 (QA · 개발 메모)

"무시" 정책: 무시 항목은 영구 삭제인지, 장비 재수집 시 재등장인지 정책 확인 필요.

06

제안 사항

기획 검토 "무시한 Pending 항목"도 History에 기록해 추적 가능해야 합니다. 실수로 무시했을 때 복구 가능하도록.
