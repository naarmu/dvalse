TC_EQP_REG_006

등록 후 Agent 설치 장비의 자동 연결

Functional Critical
01

Precondition

  1. Equipment ID EQ_NEW_CONN이 서버에 등록되었음 (Agent 미설치 상태)
  2. RMSAgent Installer와 Equipment ID·Server IP가 준비됨
  3. 장비 PC의 방화벽 및 네트워크가 서버와 통신 가능한 상태
02

Test Steps

  1. 등록 직후 Equipment Manager에서 EQ_NEW_CONN의 상태가 Offline임을 확인
  2. 장비 PC에서 RMSAgent Installer 실행
  3. Installer에서 Equipment ID EQ_NEW_CONN, Server IP 정확히 입력
  4. 설치 완료 및 Agent 자동 실행 확인
  5. RMS UI로 돌아와 EQ_NEW_CONN의 상태가 Online으로 전환되는지 확인 (최대 30초 대기)
  6. Agent Version, XEA Version 등 정보가 수집되어 표시되는지 확인
  7. Recipe 배포 가능한 상태가 되었는지 확인 (Deploy 버튼 활성화 등)
03

Test Data

Equipment: EQ_NEW_CONN Server IP: 서버 PC의 IP Agent Installer: 최신 버전
04

Expected Result

  1. Agent 설치·실행 후 30초 이내 RMS UI에서 Online으로 전환
  2. Agent Version 정보가 자동 수집되어 우측 상세에 표시됨
  3. Dashboard에도 해당 장비가 Online 상태로 동기화됨
  4. Recipe 배포 관련 기능(Deploy Recipes)에서 해당 장비가 대상으로 선택 가능
  5. 연결 성공 이력이 Equipment History에 기록됨
05

비고 (QA · 개발 메모)

Equipment ID 불일치: Installer에서 서버에 등록되지 않은 ID를 입력한 경우 연결 거부되는지 별도 확인 필요 (보안상 중요).

네트워크 포트: Agent ↔ Server 간 통신에 필요한 포트가 방화벽에서 허용되어야 함. 환경 준비 시 체크리스트 필요.

06

제안 사항

보안 강화 Installer에서 Equipment ID를 자유롭게 입력할 수 있으면, 다른 장비의 ID로 접속을 위장할 가능성이 있습니다. 서버에서 발급한 등록 토큰을 Installer에 입력하게 하는 방식이 더 안전합니다.

UX 개선 서버 측에서 장비가 처음 Online으로 전환될 때 "신규 장비 연결 성공" 알림(Toast)을 띄워주면 등록자 입장에서 확인이 쉽습니다.
