TC_EQP_REG_003

잘못된 IP 형식 입력 시 검증

Negative High
01

Precondition

  1. Vendor 또는 Admin 계정으로 로그인됨
  2. Register Equipment 화면 진입
02

Test Steps

  1. Equipment ID, Name 등 나머지 필드를 유효하게 입력
  2. IP Address 필드에 잘못된 형식의 값 입력 (여러 케이스 순차 테스트)
  3. 각 케이스마다 Save 버튼 클릭 시 반응 확인
  4. 마지막에 정상 IP 입력하여 등록 성공 확인
03

Test Data

잘못된 IP 케이스: - "192.168.1" (옥텟 부족) - "192.168.1.1.1" (옥텟 초과) - "192.168.1.300" (범위 초과) - "abc.def.ghi.jkl" (문자 포함) - "192.168.1.-1" (음수) - "" (빈 값) - "192.168.1.0" (네트워크 주소 - 정책에 따라 다름) 정상 IP: "192.168.1.151"
04

Expected Result

  1. 잘못된 형식 입력 시 Save 버튼 클릭하면 등록이 차단됨
  2. IP 필드 근처에 명확한 에러 메시지 표시 (예: "유효한 IPv4 주소 형식이 아닙니다")
  3. 입력한 다른 필드 값은 유지되어 IP만 수정 후 재시도 가능
  4. 정상 IP 입력 후 Save 시 등록 성공
  5. 가능하다면 포커스 아웃(blur) 즉시 검증하여 실시간 피드백 제공
05

비고 (QA · 개발 메모)

검증 위치: 클라이언트 측 검증 외에도 서버 측 검증이 병행되어야 함. 클라이언트 우회 시도에 대비.

IPv6 지원: 현재는 IPv4만 가정. IPv6 지원 계획이 있는지 확인.

06

제안 사항

UX 개선 IP 필드에 입력 마스크(예: "___.___.___.___" placeholder)를 적용하면 사용자가 형식을 직관적으로 인지할 수 있습니다.

기획 검토 네트워크 주소(.0), 브로드캐스트 주소(.255), 예약 범위(127.x.x.x) 등을 허용할지 말지에 대한 정책 정의가 필요합니다. 일반적으로 장비용 IP로는 허용하지 않습니다.
