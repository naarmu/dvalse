URS_SYS_SVR

서버 연결 설정

User Requirement Configuration
01

기능 개요

RMS Client가 접속할 RMS 서버의 주소(IP · 포트)를 설정하는 화면이다. Client는 이 설정을 기반으로 서버에 접속하여 로그인 인증, 데이터 조회, 작업 실행 등 모든 통신을 수행한다.

Server Setup은 Client 설치 직후 한 번 설정되며, 이후에는 서버 이전·교체 시에만 변경된다. 잘못된 설정은 RMS 사용 자체를 불가능하게 만들기 때문에 입력 검증이 중요하다.

02

사용자 니즈

주요 사용자

  • RMS 관리자 — Client 초기 설치 시 서버 정보 입력
  • 장비 엔지니어 — 서버 이전 시 신규 주소로 변경

Pain Point (RMS 도입 전)

  • 각 PC에 서버 정보를 일일이 설정 파일로 관리
  • 잘못 입력된 설정을 어떻게 수정해야 하는지 직관적이지 않음
  • 서버 변경 시 모든 Client에 동시 갱신이 어려움

기대 효과

  • UI에서 서버 정보를 직접 입력·수정 가능
  • 입력 단계에서 형식 검증으로 사용자 실수 방지
  • 설정 변경 후 즉시 연결 시도하여 결과 확인 가능
03

기능 요구사항

■ FR-1. 서버 정보 입력

  • FR-1.1 사용자는 서버 IP 주소와 포트를 입력하여 저장할 수 있어야 한다.
  • FR-1.2 입력 화면은 Client 최초 실행 시 자동 표시된다.
  • FR-1.3 로그인 화면에서 서버 설정 변경 진입점을 제공한다.
  • FR-1.4 마지막으로 저장된 서버 정보는 입력 필드에 자동 채워져 있어야 한다.

■ FR-2. 입력값 검증

  • FR-2.1 IP 주소는 IPv4 형식 검증을 거쳐야 한다. 형식 오류 시 저장을 차단한다.
  • FR-2.2 포트는 1~65535 범위 내의 숫자여야 한다. 범위를 벗어나면 저장을 차단한다.
  • FR-2.3 검증 실패 시 어느 필드의 어떤 값이 문제인지 명확한 에러 메시지를 표시한다.

■ FR-3. 서버 연결 시도

  • FR-3.1 서버 정보 저장 시 Client는 즉시 해당 서버에 연결을 시도한다.
  • FR-3.2 연결 시도 중에는 진행 상태가 표시된다.
  • FR-3.3 연결 성공 시 Login 화면으로 자동 진입한다.
  • FR-3.4 연결 실패 시 명확한 에러 메시지(타임아웃·거부·DNS 실패 등 사유별)를 표시한다.

■ FR-4. 네트워크 단절 처리

  • FR-4.1 서버 연결 중 네트워크가 단절되면 일정 횟수 자동 재시도한다.
  • FR-4.2 재시도가 모두 실패하면 사용자에게 알리고 Server Setup 화면으로 안내한다.
  • FR-4.3 사용자가 수동으로 재연결을 시도할 수 있는 버튼을 제공한다.

■ FR-5. 설정값 보존

  • FR-5.1 저장된 서버 정보는 Client를 재시작해도 유지되어야 한다.
  • FR-5.2 설정값은 Client가 위치한 PC의 사용자별 영역에 보관된다.
04

비기능 요구사항

■ NFR-1. 보안

  • NFR-1.1 Client와 서버 간 통신은 TLS로 암호화된 채널 위에서 수행되어야 한다.
  • NFR-1.2 서버 인증서가 유효하지 않은 경우 사용자에게 경고를 표시하고 연결 진행 여부를 확인한다.

■ NFR-2. 사용성

  • NFR-2.1 입력값 검증 결과는 사용자가 입력하는 도중 실시간으로 표시되어야 한다.
  • NFR-2.2 연결 실패 시의 에러 메시지는 사용자가 다음 행동을 결정할 수 있도록 명확해야 한다.

■ NFR-3. 성능

  • NFR-3.1 정상 환경에서 서버 연결 시도부터 결과 표시까지 5초 이내.
  • NFR-3.2 연결 시도 타임아웃은 10초 이내로 제한한다.
05

제약사항 · 정책

  • 정책 Server Setup은 Client 최초 실행 시 자동 표시한다. 사용자가 별도로 찾을 필요 없도록 한다.
  • 정책 연결 실패 메시지는 사유별로 구분한다(타임아웃 / 거부 / DNS 실패 등). 사용자가 다음 행동을 결정할 수 있게 한다.
  • 정책 서버 정보 저장 시 즉시 연결을 시도하여 결과를 확인하게 한다. 잘못된 설정 후 다음 단계에서 실패하는 흐름을 방지한다.
  • 제약 Client–서버 통신은 TLS 암호화 채널 위에서만 허용한다.
  • 제약 연결 타임아웃은 10초 이내로 제한한다.
06

관련 Test Cases

이 URS의 요구사항을 검증하는 QA 테스트 케이스 5건:

  • TC_SYS_SVR_001 — Server Setup 팝업에서 서버 주소 입력 · 저장 (FR-1)
  • TC_SYS_SVR_002 — 잘못된 IP · 포트 입력 시 연결 실패 처리 (FR-2, FR-3.4)
  • TC_SYS_SVR_003 — 서버 연결 성공 시 메인 진입 (FR-3.1, FR-3.3)
  • TC_SYS_SVR_004 — 서버 연결 중 네트워크 단절 재시도 (FR-4)
  • TC_SYS_SVR_005 — Server Setup 설정값 재접속 시 유지 (FR-5)
