TC_RCP_MGR_006

Recipe 버전 목록 표시

Functional Critical
01

Precondition

  1. 여러 버전을 가진 Recipe "213"이 존재 (v1, v2)
  2. 단일 버전만 있는 Recipe도 함께 존재
02

Test Steps

  1. Recipe "213"의 펼침 토글(▶) 클릭
  2. 하위에 버전 목록(v1, v2)이 표시되는지 확인
  3. 각 버전에 번호와 수집/등록 일시가 함께 표시되는지 확인
  4. Master Recipe로 지정된 버전의 표시 방식 확인 (아이콘/배지 등)
  5. 단일 버전 Recipe도 동일하게 v1 표시되는지 확인
03

Test Data

Recipe 213: - v2 (2026-04-13 11:17) [Master] - v1 (2026-04-10 13:45)
04

Expected Result

  1. 버전 목록이 Recipe 하위 노드로 정상 펼침 표시
  2. 각 버전에 v{번호}와 일시(yyyy-MM-dd HH:mm)가 함께 표시
  3. Master 버전은 별도 아이콘/배지로 시각적 구분
  4. 버전 선택 시 중앙/우측 상세 영역이 해당 버전 기준으로 표시
  5. 접힘 상태 저장(선택): Recipe별 펼침/접힘 상태가 세션 내에서 유지되면 좋음
05

비고 (QA · 개발 메모)

버전 번호 규칙: v1, v2 처럼 순차 증가인지, 버전 이름 커스텀 가능한지 확인. slide-27은 v1, v2 순차.

Master 표시: Master Recipe 지정은 Manage Recipe Versions 화면에서 이뤄지는데, 그 결과가 Recipe Manager 트리에도 반영되는지 확인 필수.

06

제안 사항

정보 구조 Master 버전은 ⭐ 또는 M 배지로 한눈에 구분 가능해야 합니다. 반도체 공정에서 잘못된 버전 배포는 치명적이므로 시각적 강조가 중요합니다.

UX 개선 버전 목록에서 각 버전의 주요 변경 요약(diff summary)을 mouseover 시 tooltip으로 제공하면 버전 선택 전에 빠른 판단이 가능합니다.
