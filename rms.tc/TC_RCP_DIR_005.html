TC_RCP_DIR_005

동일 이름 Recipe 업로드 시 버전 처리

Functional High
01

Precondition

  1. 서버에 Recipe "213" v1이 이미 존재
  2. 로컬에 동일 이름의 .psr이 있음 (내용은 일부 다름)
02

Test Steps

  1. 동명의 "213.psr" 업로드 시도
  2. 동명 파일 감지 시 UI 동작 확인 (덮어쓰기? 새 버전? 에러?)
  3. 처리 정책에 따른 결과 확인
03

Test Data

Existing: Recipe 213 v1 / Uploading: 213.psr (다른 내용)
04

Expected Result

  1. 동명 파일 감지 시 사용자에게 옵션 제공:
    • ① 새 버전으로 추가 (권장)
    • ② 기존 버전 덮어쓰기
    • ③ 취소
  2. 선택한 동작에 따라 Recipe 버전 트리 업데이트
  3. 이력에 선택 사항 기록
05

비고 (QA · 개발 메모)

정책 결정 필수: 자동 덮어쓰기는 데이터 손실 위험이 큼. 사용자 확인이 필요한 영역.

06

제안 사항

데이터 무결성 동명 파일 업로드 시 기본 동작은 "새 버전 추가"로 설정하는 것이 안전합니다. 덮어쓰기는 실수로 인한 원본 손실 위험이 큽니다.
