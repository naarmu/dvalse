TC_EQP_MGR_004

장비 목록 필터 · 검색 동작

Functional High
01

Precondition

  1. Equipment Manager 화면에 최소 10대의 장비가 서로 다른 이름·그룹으로 등록되어 있음
  2. Online 장비와 Offline 장비가 혼재
02

Test Steps

  1. 검색창에 장비 이름 일부(예: "NX10") 입력
  2. 실시간 필터링 결과 확인
  3. 검색어 지움 → 전체 목록 복귀 확인
  4. 상태 필터(Online/Offline 등)가 제공되는 경우 각 필터 선택 후 결과 확인
  5. 검색 + 필터 동시 적용 시 결과 확인
03

Test Data

Search Term: "NX10" (EQ_001, EQ_003에 해당) Status Filter: Online / Offline Group Filter: GROUP_A, GROUP_B
04

Expected Result

  1. 검색어 입력 시 장비명·Equipment ID 중 일부라도 일치하는 장비만 목록에 표시
  2. 검색어가 지워지면 전체 목록 즉시 복귀
  3. 상태 필터는 배타적(단일) 또는 다중 선택으로 정상 작동
  4. 검색어 + 필터 동시 적용 시 AND 조건으로 결과 좁혀짐
  5. 결과가 0건일 때 "일치하는 장비가 없습니다" 등 Empty State 메시지 표시
05

비고 (QA · 개발 메모)

검색 대상 필드: Equipment Name만인지, Equipment ID와 Description까지 포함인지 명세 확인 필요. 사용자가 IP로 검색하려 할 가능성도 고려.

대소문자 구분: 검색 시 대소문자를 구분하지 않아야 함 — 반드시 확인.

06

제안 사항

UX 개선 검색창에 최근 검색어 히스토리 또는 자동완성을 제공하면 반복 작업이 편해집니다. 장비 운영팀은 특정 장비 ID 패턴을 반복적으로 검색하는 경향이 있습니다.

기획 검토 검색 디바운스 타이밍(예: 300ms)이 설정되어 있는지 확인 필요합니다. 빠르게 타이핑할 때 매 키 입력마다 검색 API를 호출하면 성능에 부담을 줄 수 있습니다.
