TC_EQP_REG_002

중복 장비 ID 등록 차단

Negative High
01

Precondition

  1. RMS Client가 실행되고 서버에 연결되어 있음
  2. Vendor 또는 Admin 계정으로 로그인됨
  3. 서버에 이미 등록된 장비 ID EQ_001이 존재함 (사전 세팅)
02

Test Steps

  1. Equipment Manager 화면에서 Add Equipment 버튼 클릭
  2. Equipment ID 필드에 EQ_001 입력 (기존과 중복되는 값)
  3. 나머지 필수 필드(IP, 장비명 등)를 유효한 값으로 입력
  4. Save (또는 등록) 버튼 클릭
  5. 시스템 응답 및 폼 상태 확인
03

Test Data

Equipment ID: EQ_001 (중복) IP Address: 192.168.1.100 Name: Test Equipment Group: GROUP_A
04

Expected Result

  1. 등록이 차단되고 명확한 에러 메시지가 표시됨 (예: "이미 존재하는 Equipment ID입니다")
  2. 서버 DB에 중복 레코드가 생성되지 않음 (사후 조회로 확인)
  3. 입력된 폼 값은 초기화되지 않고 유지되어 사용자가 ID만 수정해 재시도할 수 있음
  4. Equipment ID 필드에 포커스가 자동 이동되거나 에러 표시가 명확해야 함
  5. 이 실패 시도가 RMS Log 또는 Server Log에 기록됨(예상)
05

비고 (QA · 개발 메모)

검증 경계: 중복 검사가 클라이언트 측 only / 서버 측 only / 양쪽 모두 중 어디서 이루어지는지 확인 필요. 권장은 서버 측 검증이며, 클라이언트 즉시 피드백은 UX 보조로만 활용해야 함.

에러 문구 일치: 기획 UX 표준 문구가 확정되어 있다면 해당 문구와 정확히 일치해야 함. 현재 가정: "이미 존재하는 Equipment ID입니다".

06

제안 사항

UX 개선 Equipment ID 입력 필드에서 포커스 아웃(blur) 시점에 실시간 중복 검사를 수행하여 Save 버튼을 누르기 전에 피드백을 주는 것을 권장드립니다. 장비 등록 중 실수로 중복 입력하는 경우, Save까지 간 후 다시 수정하는 것보다 즉시 알려주는 쪽이 UX가 훨씬 낫습니다.

기획 검토 "Equipment ID"의 명명 규칙(naming convention)이 정의되어 있는지 확인이 필요합니다. (예: 대문자·숫자·언더스코어만 허용, 최대 N자 등) 정의되어 있다면 별도 TC(잘못된 포맷 차단)를 추가해야 합니다.

데이터 무결성 삭제된 장비 ID를 재사용 가능한지에 대한 정책 확인이 필요합니다. 재사용 허용 시 해당 ID로 축적된 History가 새 장비에 섞여 보일 위험이 있습니다.
