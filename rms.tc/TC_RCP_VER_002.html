TC_RCP_VER_002

특정 버전을 Master Recipe로 지정

Functional Critical
01

Precondition

  1. Recipe "213"이 v1, v2 두 버전을 가지며 아직 Master가 지정되지 않음 (또는 v1이 현재 Master)
02

Test Steps

  1. Recipe "213" 선택
  2. v2 행에서 "Set as Master" 버튼 또는 체크박스 클릭
  3. 확인 팝업(있는 경우) 확인 후 진행
  4. v2가 Master로 지정되었는지 확인
  5. Recipe Manager 트리와 Dashboard에서도 v2가 Master로 표시되는지 확인
03

Test Data

Recipe: 213 Before: v1 (Master), v2 After: v1, v2 (Master)
04

Expected Result

  1. v2가 Master로 지정되고 UI에 즉시 반영
  2. 이전 Master(v1) 지정은 자동 해제 (단일 Master 원칙)
  3. 지정 이력이 Recipe History에 기록 (변경자, 시각, 이전 Master, 새 Master)
  4. Recipe Manager 및 Dashboard의 Master 표시도 즉시 갱신
05

비고 (QA · 개발 메모)

Master 정의: Master Recipe는 "기준 버전"으로 Deploy 기본값, Compare 기준으로 사용됨. 개념 명확히 문서화 필요.

06

제안 사항

UX 개선 Master 변경 시 변경 사유 입력을 선택 필드로 받으면 이력 추적에 도움이 됩니다. 필수는 아니지만 권장.

보안 Master 변경은 중요한 변경 이벤트입니다. 권한을 Admin 이상으로 제한하는 것도 고려해주세요.
