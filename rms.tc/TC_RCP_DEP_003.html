TC_RCP_DEP_003

다수 장비 동시 배포 (4대 이상)

Functional Critical
01

Precondition

  1. Online 장비 4대 이상 연결됨 (EQ_001~EQ_004)
  2. 배포할 Recipe/버전 선택 완료
02

Test Steps

  1. 대상 장비 영역에서 EQ_001, 002, 003, 004 모두 선택 (또는 "전체 선택")
  2. Deploy 버튼 클릭
  3. 확인 팝업에 대상 장비 목록(4대) 명시 확인 후 진행
  4. 배포 진행률 표시 확인 (장비별 개별 상태)
  5. 모든 장비 배포 완료 후 개별 결과 확인
03

Test Data

Targets: EQ_001, EQ_002, EQ_003, EQ_004 (모두 Online) Recipe: 213 v2
04

Expected Result

  1. 4대 장비 동시 선택 가능
  2. 병렬 배포 진행 (직렬이어도 무방하나 병렬이 성능상 유리)
  3. 장비별 진행 상태가 개별 표시 (예: EQ_001: 80%, EQ_002: 완료, ...)
  4. 모든 장비 배포 완료 시 종합 결과 요약 표시 (성공 N건 / 실패 M건)
  5. Deployment Log에 각 장비별로 개별 이력 기록
05

비고 (QA · 개발 메모)

네트워크 부하: 동시 배포 시 서버→장비 파일 전송이 병렬로 발생. 서버 네트워크 대역폭 및 동시 연결 수 한계 확인 필요.

06

제안 사항

성능 동시 배포 최대 개수에 대한 제한이 필요할 수 있습니다. 예: "동시 최대 10대". 서버 부하 및 실패 시 복구 관점에서 중요.

UX 개선 배포 진행 중에도 화면을 벗어날 수 있어야 합니다. 백그라운드 진행 + 상단 알림 아이콘으로 진행률 표시 권장.
