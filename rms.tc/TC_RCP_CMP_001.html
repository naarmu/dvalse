TC_RCP_CMP_001

Compare Recipes 화면 진입 및 대상 선택 UI

Functional Critical
01

Precondition

  1. Recipe가 2개 이상 서버에 등록된 상태
  2. Compare Recipes 화면으로 진입 가능
02

Test Steps

  1. Recipe 메뉴 → Compare Recipe 클릭
  2. 화면 진입 후 비교 대상 선택 UI 확인
  3. 첫 번째 대상으로 Recipe A / Version 선택
  4. 두 번째 대상으로 Recipe B / Version 선택
  5. 선택 후 파라미터 비교 결과 영역이 활성화되는지 확인
03

Test Data

Recipe A: 213 / v1 Recipe B: SDC_A1WPP_GZ3_LENS_ADI / v1
04

Expected Result

  1. 화면 진입 후 비교 대상 슬롯(2개 이상)이 표시됨
  2. 각 슬롯에서 Recipe 선택 시 버전을 지정할 수 있음
  3. 2개 이상 선택되면 "비교 실행" 버튼 활성화 또는 즉시 비교 결과 표시
  4. 잘못된 조합(같은 Recipe 같은 버전 2번 등)은 차단 또는 경고
05

비고 (QA · 개발 메모)

진입 경로: Recipe Manager에서 Recipe 선택 후 "Compare" 버튼으로 진입하는 경로도 있는지 확인 필요. 그 경우 첫 번째 슬롯에 자동 입력되면 UX 좋음.

06

제안 사항

UX 개선 Compare Recipes 진입 시 Recipe Manager에서 선택했던 Recipe가 첫 슬롯에 자동 입력되는 플로우를 권장드립니다. 반복 선택을 줄일 수 있습니다.

기획 검토 "최근 비교 이력"을 저장해 원클릭 재실행을 지원하면 반복적인 비교 작업이 편해집니다.
