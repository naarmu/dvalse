TC_E2E_004

Direct Upload로 Recipe 등록 → Master 지정 → 그룹 장비에 일괄 배포

Functional Critical
01

Precondition

  1. 로컬 PC에 유효한 .psr 파일 준비
  2. 장비 그룹 "9F [DRD]"에 장비 4대 소속, 모두 Online
  3. Engineer 이상 권한
02

Test Steps

  1. Recipe Manager에서 Upload 버튼으로 로컬 .psr 업로드 (TC_RCP_DIR_001 연계)
  2. 업로드된 Recipe가 Recipe Manager 목록에 표시 확인
  3. 출처가 "Direct Upload"로 표시되는지 확인 (TC_RCP_DIR_004 연계)
  4. Manage Recipe Versions에서 해당 Recipe를 Master로 지정
  5. Deploy Recipes에서 장비 그룹 "9F [DRD]" 선택 (TC_RCP_DEP_004 연계)
  6. 그룹 내 모든 장비에 일괄 배포 실행
  7. 배포 진행 상태 및 장비별 결과 확인 (TC_RCP_DEP_007, 008 연계)
  8. Deployment Log에서 4대 장비 각각의 배포 이력 확인
03

Test Data

Direct Upload: production_recipe.psr Group: 9F [DRD] (4대 장비: EQ_001 ~ EQ_004)
04

Expected Result

  1. 로컬 파일이 RMS 내부 Recipe로 정상 등록
  2. Master 지정 후 그룹 일괄 배포가 병렬로 수행
  3. 4대 장비 모두 배포 완료, Deployment Log에 4건 기록
  4. Direct Upload Recipe도 일반 Recipe와 동일 기능 사용 가능
05

비고 (QA · 개발 메모)

Phase 2 신규: Direct Upload는 장비 외부에서 Recipe를 도입하는 경로. 공정 표준화 팀이 중앙에서 Recipe를 관리할 때 유용.

06

제안 사항

보안 Direct Upload 경로는 악성 파일 유입 가능성이 있는 유일한 경로입니다. 업로드 시 파일 검증 + 장비 배포 전 추가 확인 팝업을 권장드립니다.

UX 개선 Direct Upload한 Recipe를 배포할 때 "이 Recipe는 Direct Upload된 것입니다. 검증되었나요?" 같은 추가 확인 팝업이 안전합니다.
