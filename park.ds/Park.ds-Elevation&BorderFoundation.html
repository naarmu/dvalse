§ 0

철학 — border가 기본이다

tl;dr

Park.ds의 계층감은 border로 먼저 세운다. shadow는 떠 있어야 하는 것 — modal, dropdown, 그리고 카드·버튼의 아주 약한 분리감 — 에만 쓴다. dark에서는 shadow 대신 surface tint(면 밝기 차)와 border가 계층을 만든다.

§ principle · 1
flat-first, shadow는 overlay 전용

전역 배경 위에 놓이는 카드/패널은 border만 쓴다. shadow는 popover/modal처럼 진짜로 "떠야 할 것"에만.

§ principle · 2
border는 두께 아닌 색으로

1px 한 종류. 위계는 border.subtle / default / strong / focus 4단계로 색만 바꾼다. 2px는 focus outline에만.

§ principle · 3
dark에서 shadow는 거의 쓰지 않는다

dark는 빛 자체가 줄어드는 공간. 떠 있음을 표현하려면 tint 레벨과 border를 올린다. popover/modal만 예외로 깊고 진한 그림자 1개.

§ principle · 4
focus는 outline, shadow 아님

layout 흔들림을 피하기 위해 outline: 2px + outline-offset: 0. border는 brand 색으로 바꾸고, outline은 반투명 ring.

§ principle · 5
inset stroke는 제한적 보조

input의 내부 도트(1px inset), 스위치 트랙 같이 누름 감이 필요한 곳만. 장식으로 쓰지 않는다.

§ principle · 6
한 화면에 elevation은 2단계까지

같은 영역 안에서 raised → popover → modal이 겹치지 않게. 겹치면 정보 위계가 무너진다.

§ 3

Shadow 스케일 — 4단계, hybrid (crisp + soft)

§3

low-elevation(raised, popover)은 crisp(작은 blur + 뚜렷한 line). modal은 soft(넓은 blur + 깊은 offset). 각 단계 모두 1px border를 함께 사용.

flat
shadow.flat
depth 0 · border only
border: 1px solid #E3E6EB;
→ 패널, 일반 section, body-level card. 대부분 여기 속함.
raised
shadow.raised
depth 1 · hairline lift
0 1px 2px rgba(18,28,40,0.06), 0 0 0 1px rgba(18,28,40,0.05)
→ dashboard KPI card, hover된 list item. light 전용 (dark는 border만).
popover
shadow.popover
depth 2 · crisp
0 2px 4px rgba(18,28,40,0.08), 0 4px 12px rgba(18,28,40,0.08), 0 0 0 1px rgba(18,28,40,0.06)
→ dropdown, tooltip, combobox, right-click menu.
modal
shadow.modal
depth 3 · soft
0 2px 6px rgba(18,28,40,0.08), 0 20px 48px rgba(18,28,40,0.22)
→ dialog, confirm, sheet. 반드시 backdrop과 함께.

shadow.flat

→ dropdown 등. light보다 더 진하고 단순한 shadow 하나.
modal
shadow.modal
deep
0 8px 32px rgba(0,0,0,0.50)
→ dialog. 배경을 강하게 어둡게.
* 왜 hybrid인가 — popover는 "떠 있지만 가깝다"(crisp), modal은 "작업이 일시 정지됐다"(soft/deep). 둘의 시각 언어는 달라야 한다.
* 왜 dark에서 shadow를 줄이나 — 어두운 면 위의 소프트 그림자는 잘 안 보이고, 보이면 더러워 보인다. 대신 surface tint(면 밝기)와 border.strong으로 계층.
* alpha는 검정(#000)이 아닌 ink-dark(#121C28) — 순검정 그림자는 회색 UI 위에서 tint가 낯설다.

§ 6

미정 · Tweak 후보 — 실제 적용하며 정할 것

§6

추후 결정 (Park.ds v0.2)

  • surface tint 레벨 — dark에서 shadow 자리를 대체할 tint 단계를 몇 개로 할지. 현재 암묵적으로 surface.base / raised 2단계. KPI card / popover / modal 까지 고려해 3단계(base / raised / overlay)로 확장할지 실제 WLI 화면에 적용하며 결정.
  • shadow alpha base color — 현재 rgba(18,28,40,…). 프로젝트마다 브랜드 톤이 실어진 그림자 색(약간의 blue tint)을 쓸지, neutral로 유지할지.
  • raised card hover — hover 시 shadow.raised 그대로 둘지, 한 단 더 올릴지. 계측 SW는 hover 빈도가 낮아 유지하는 쪽이 유력.
  • border.subtle in dark — 현재 #22272E(background 바로 위). 표에서 row divider가 거의 안 보이는데 의도된 건지, 혹은 #2B313A(=default)로 통일할지.
  • focus ring 두께 — 2px 유지 vs 3px (접근성 우선) — WLI는 마우스 중심이라 2px 충분할 것으로 보임.
  • modal border in dark — #3A414C (strong) 유지 vs #4B5564 (더 밝게) — 깊은 그림자와 함께 "떠 있음"을 강조할지.
