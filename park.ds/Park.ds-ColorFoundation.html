§0

설계 의도 세 문장으로 요약

Park.ds의 색은 산업용·일관·명료라는 세 원칙에 복무한다. 화면의 90% 이상은 저채도 중립색이 차지하며, 채도는 강조(Accent)·상태(Status)·장비(Equipment)의 세 자리에서만 허용된다. 팔레트는 #30A1D2를 축(L 0.67 / C 0.12 / h 232°)으로 OKLCH 공간에서 수학적으로 확장되어, Light와 Dark가 동일한 Semantic Slot을 공유한다.

§3

Equipment State Colors L2 · 장비 상태 전용 확장 토큰

장비 상태 표시는 일반 Status와 의도적으로 분리한다. 이유:

  • 장비 상태는 수 시간 이상 화면에 노출되므로 과한 고채도는 피로도 유발
  • Success/Error 같은 일반 UI 상태와 의미 충돌 방지 (Running ≠ Success)
  • 도트·라벨·배경 3가지 표현이 일관되게 쓰임
Running
equip.running · #2E8B57
Idle
equip.idle · #8A94A3
Paused
equip.paused · #C58A1A
Calibrating
equip.calibrating · #7E5BC4
Error
equip.error · #C23A3A
Offline
equip.offline · #5A6370

* Running 도트만 pulse 애니메이션 허용 (1.6s ease-out). 나머지는 정적. 가독성과 과도한 시각 자극 억제.

§4

DevExpress Theme Designer 매핑 실제 구현 레이어

DevExpress의 Palette Color / Common Color 슬롯은 의미 기반으로 이미 정의되어 있다. Park.ds는 이 슬롯을 권위로 인정하고, §2 Semantic Token을 그 위에 매핑한다. 아래 매핑은 DevExpress Theme Designer의 "My Theme" 편집 화면에서 그대로 입력 가능한 형태다.

DevExpress Slot Park.ds Semantic Token 비고
Palette.Accent accent.default Button primary, Selection highlight 등 제품 전반 Accent
Palette.Background surface.base DocumentGroup, MainWindow 바탕
Palette.SecondaryBackground surface.panel Panel, CardView, NavigationBar 등
Palette.Foreground text.primary 기본 텍스트
Palette.SecondaryForeground text.secondary 서브 라벨, 캡션
Palette.HoverBackground state.hover Accent 10% 반투명 권장
Palette.SelectedBackground state.selected Accent 18% 반투명 권장
Palette.Border border.default 모든 1px 구분선
Common.Error status.error Validation, Critical alarm
Common.Warning status.warning 주의 레벨 알림

* Equipment State 계열(§3)은 DevExpress 기본 슬롯에 대응이 없으므로 커스텀 Brush Resource로 따로 정의한다: ParkEquip.RunningBrush, ParkEquip.IdleBrush 등.

§5

피드백 요청 이 초안에서 결정이 필요한 지점

Figma 팔레트와 대조하며 확인 부탁

  1. Gray의 온도 — h 232° 방향으로 살짝 cool한 neutral로 설계했다. Figma 것과 비교해 더 중립(h 0° pure gray)으로 가는 게 나을지, 유지할지.
  2. Blue Ramp의 50/900 끝단 — 50은 거의 흰색에 근접, 900은 잉크에 근접. 실제 사용 케이스(배너 배경, 헤더 강조 등)를 생각했을 때 스텝이 부족/과한지.
  3. Amber vs Orange — Warning을 h 75°(Amber, 녹황)으로 뒀는데, 산업 관행상 h 55°(Orange 계열)이 더 관습적이다. 선호 방향?
  4. Dark mode의 Accent — Light의 #30A1D2를 그대로 쓰면 채도가 너무 강해 눈이 피로해, Dark에선 L 0.76 / C 0.10 으로 완화(#45B4E2)했다. 이 톤이 맞는지.
  5. Equipment State 슬롯 추가 — 현재 6종(Running / Idle / Paused / Error / Calibrating / Offline)이다. 제품에 실재하는 상태 중 빠진 것이 있는지. (예: Warming-up, Homing, Aborting, Maintenance 등)
