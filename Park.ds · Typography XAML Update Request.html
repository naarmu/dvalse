개발진 요청 메시지 이 섹션 복사해 붙여 넣어 전달

REQUEST
저번에 드렸던 Pancras*TextBlock Style 부분을 아래 §3의 XAML 블록으로 통째로 교체해 주세요.

Park.ds v0.1 Typography Foundation을 확정하면서 기존 Stylecode와 일부 조정이 있었습니다. 구조(네이밍/계층)는 그대로 유지하되, Weight와 일부 Size·Digital FontFamily가 바뀝니다.

  1. 네이밍·계층은 유지 — Pancras*TextBlock prefix와 Display/Headline/Title/Body/Label/Annotation/Digital 7계층 그대로.
  2. Weight 단순화 — 앞으로는 Regular(400) / Medium(500) / SemiBold(600) / Bold(700) 4단계만 사용합니다. Thin·ExtraLight·Light는 사용하지 않습니다.
  3. Label의 Uppercase 규칙 제거 — TextBlockAssist.CharacterCasing="Upper" 세터를 빼 주세요.
  4. Digital FontFamily 교체 — DSEG7ModernFont → DSEG14ClassicFont. 14-세그먼트가 알파벳까지 표현 가능해 확장성이 좋습니다.
  5. 오타 수정 — PancrasDigitalMediumTextBlock의 FontSize="13." → "13".
  6. Display Large FontSize 조정 — 56 → 48. (장비 판독 대형 수치는 Display.Medium 이하 선에서 운용되며, 56은 거의 쓰이지 않아 스케일 일관성을 위해 축소)

변경 전/후 대조와 배경은 §1~§2, 최종 교체 코드는 §3에 있습니다.

변경 원칙 요약

  • Display/Headline 계열: 기존에는 Thin~Light(100~300)로 Apple 감성을 노렸으나, 화면이 작은 장비 운용 환경(FHD 24")에서는 저굵기가 가독성을 해침. Regular/SemiBold로 차분하게 정리.
  • Body/Title는 기존 그대로 유지(충돌 없음).
  • Label의 Uppercase 강제는 한/영 혼용 환경에서 어색하므로 제거. 강조는 위치·색으로 주는 것이 Park.ds 원칙.
  • Annotation의 최소 크기(9px)는 경고 캡션·각주 전용으로 유지. UI 본문에는 사용 금지.
  • Digital은 DSEG14로 교체. 숫자뿐 아니라 V · Hz · FPS 등 단위 표기까지 같은 서체로 가능해짐.

개발진 측 체크 포인트

  • DSEG14ClassicFont MarkupExtension 준비 — 기존 {font:DSEG7ModernFont}와 같은 방식으로 DSEG14ClassicFont를 등록해야 합니다. 폰트 파일(OFL)은 공유해드릴 수 있으니 요청해 주세요.
  • TextBlockAssist.CharacterCasing 유무 — Label 3개 스타일에서 세터 자체를 제거해 주세요(값만 None으로 바꾸는 게 아니라).
  • Pretendard Variable FontFamily 바인딩 — 기본 FontFamily는 별도 Base Style(예: PancrasBaseTextBlock)에서 상속받는 구조로 가정했습니다. 그 구조가 유효한지 확인 부탁.
